# განმარტება — 390. ნაკლული და ნამეტი რიცხვები

## მიდგომა

ამოცანა საკმაოდ მარტივია: უბრალოდ უნდა ვიპოვოთ რიცხვის საკუთარი გამყოფები და მათი ჯამი შევადაროთ თავად რიცხვს. რადგან n მაქსიმუმ 10⁵-ია, O(n) სირთულის ციკლი იდეალურად მუშაობს. 1-დან n/2-მდე ყველა რიცხვს ვამოწმებ `x % p == 0` პირობით და თუ გამყოფია, ვუმატებ `k` ცვლადს.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- გამყოფების ძებნა n/2-ზე მეტ რიცხვებში აზრს კარგავს, ამიტომ ციკლის საზღვარი `x/2` სრულიად საკმარისია.
- 6-ის მსგავსი რიცხვები, სადაც ჯამი თვითონ რიცხვის ტოლია, არც `defective` არის და არც `excessive`, რაც პროგრამაში `else`-ით მარტივად იჭერს.

## ხშირი შეცდომები

- ციკლის 0-დან დაწყება, რაც ნულზე გაყოფის შეცდომას გამოიწვევს.
