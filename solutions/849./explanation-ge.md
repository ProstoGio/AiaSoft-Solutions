# განმარტება — 849. სად არის მინიმუმი?

## მიდგომა

ამ ამოცანაში ველოსიპედის გამოგონება არ გვჭირდება. უბრალოდ ვინახავთ პირველ ელემენტს როგორც მინიმუმს და მის ინდექსს, მერე კი თითოეულ მომდევნო რიცხვს ვამოწმებთ. როგორც კი იმაზე პატარა რიცხვს ვნახულობთ, ვიმახსოვრებთ ახალ მნიშვნელობას და მის ინდექსსაც. ერთი ციკლი საკმარისია, რომ ყველაფერი დავითვალოთ.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- არ ღირს რთული მეთოდების ძებნა, ჩვეულებრივი `if` და ერთი `for` ციკლი აქ ყველაზე ოპტიმალურია.
- ინდექსის ცალკე ცვლადში შენახვა გვაცილებს თავიდან მეორედ მასივში სირბილს.

## ხშირი შეცდომები

- ინდექსების არევა, როცა მასივს 1-დან იწყებენ დათვლას, მაშინ როცა C++-ში 0-დან იწყება.
- მინიმუმის საწყის მნიშვნელობად 0-ის მიცემა - თუ მასივში მარტო დადებითი რიცხვებია, ეს ლოგიკას მთლიანად ანგრევს.
