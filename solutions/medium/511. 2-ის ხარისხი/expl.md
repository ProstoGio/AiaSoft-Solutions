# ამოხსნის ახსნა

ამოცანაში მოცემულია ერთი ნატურალური რიცხვი `N`.

უნდა დავადგინოთ, შესაძლებელია თუ არა მისი ჩაწერა სახით:

```text
2^k
```

სადაც:

```text
k ≥ 0
```

და `k` მთელი რიცხვია.

---

# მაგალითები

## 1

```text
1 = 2^0
```

პასუხი:

```text
YES
```

---

## 8

```text
8 = 2^3
```

პასუხი:

```text
YES
```

---

## 64

```text
64 = 2^6
```

პასუხი:

```text
YES
```

---

## 12

```text
12 ≠ 2^k
```

პასუხი:

```text
NO
```

---

# მთავარი იდეა

თუ რიცხვი 2-ის ხარისხია, მაშინ მისი 2-ზე გაყოფა შეგვიძლია მანამ, სანამ 1-მდე არ მივალთ.

მაგალითად:

```text
16
```

---

ვყოფთ 2-ზე:

```text
16 → 8 → 4 → 2 → 1
```

ბოლოს მივიღეთ `1`, ამიტომ:

```text
16 = 2^4
```

---

ახლა განვიხილოთ:

```text
12
```

---

ვყოფთ 2-ზე:

```text
12 → 6 → 3
```

აქ მივიღეთ კენტი რიცხვი, რომელიც 1 არ არის.

ეს ნიშნავს, რომ:

```text
12
```

ვერ იქნება 2-ის ხარისხი.

---

# რეკურსიული ფუნქცია

პროგრამაში გამოყენებულია ფუნქცია:

```cpp
long long power(long long n)
```

რომელიც ამოწმებს, მივალთ თუ არა საბოლოოდ `1`-მდე.

---

## პირველი შემთხვევა

```cpp
if(n%2!=0 && n!=1){
    return 0LL;
}
```

თუ რიცხვი:

- კენტია;
- მაგრამ 1 არ არის;

მაშინ ის ვერ იქნება 2-ის ხარისხი.

---

მაგალითად:

```text
3
5
7
9
15
```

ყველა ეს რიცხვი მაშინვე გვაძლევს პასუხს:

```text
NO
```

---

## მეორე შემთხვევა

```cpp
else if(n%2!=0 && n==1){
    return 1LL;
}
```

თუ მივედით:

```text
n = 1
```

მაშინ წარმატებით გავიარეთ ყველა გაყოფა.

ეს ნიშნავს, რომ საწყისი რიცხვი ნამდვილად იყო:

```text
2^k
```

და ვაბრუნებთ:

```cpp
1
```

რაც წარმატების ნიშანია.

---

## მესამე შემთხვევა

```cpp
else if(n%2==0){
    power(n>>1);
}
```

თუ რიცხვი ლუწია, ვაგრძელებთ შემოწმებას.

---

# რატომ გამოიყენება `n >> 1` ?

ბიტური ოპერაცია:

```cpp
n >> 1
```

ნიშნავს:

```text
n / 2
```

(მთელი გაყოფა)

---

მაგალითად:

```text
16 >> 1 = 8
8 >> 1 = 4
4 >> 1 = 2
2 >> 1 = 1
```

---

ამგვარად რეკურსია ეტაპობრივად ამცირებს რიცხვს.

# სირთულე

ყოველ ნაბიჯზე რიცხვი ორჯერ მცირდება.

---

### დროითი სირთულე

```text
O(log N)
```

---

### მეხსიერების სირთულე

```text
O(log N)
```

რეკურსიული გამოძახებების გამო.

# მნიშვნელოვანი შენიშვნა

შენი კოდის იდეა სწორია: ლუწ რიცხვებს მუდმივად ყოფ 2-ზე და ამოწმებ, საბოლოოდ 1-მდე მიხვალ თუ არა.

თუმცა ტექნიკურად ფუნქციაში

```cpp
power(n>>1);
```

უნდა ეწეროს

```cpp
return power(n>>1);
```

რადგან რეკურსიული გამოძახების შედეგი უნდა დაბრუნდეს უკან.

ანუ ეს ნაწილი:

```cpp
else if(n%2==0){
    power(n>>1);
}
```

უფრო სწორად ასე უნდა იყოს:

```cpp
else if(n%2==0){
    return power(n>>1);
}
```

წინააღმდეგ შემთხვევაში ფუნქციის ქცევა ფორმალურად განუსაზღვრელია, მიუხედავად იმისა, რომ ბევრ ტესტზე შეიძლება მაინც სწორად იმუშაოს.

# შეჯამება

რიცხვი წარმოადგენს 2-ის არაუარყოფით მთელ ხარისხს მაშინ და მხოლოდ მაშინ, როცა:

```text
მისი 2-ზე გაყოფა შეგვიძლია რამდენჯერმე
და საბოლოოდ ზუსტად 1-ს მივიღებთ.
```

პროგრამა სწორედ ამ იდეას იყენებს:

1. თუ რიცხვი ლუწია, ვყოფთ 2-ზე;
2. თუ მივედით 1-მდე, პასუხია `YES`;
3. თუ მივედით კენტ რიცხვამდე, რომელიც 1 არ არის, პასუხია `NO`.

ამგვარად `O(log N)` დროში ვადგენთ არის თუ არა მოცემული რიცხვი 2-ის ხარისხი.
