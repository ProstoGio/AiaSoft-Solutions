# ამოხსნის ახსნა

ამოცანაში საჭიროა დავითვალოთ, რამდენი `N` სიგრძის ორობითი მიმდევრობა არსებობს, რომელიც შედგება მხოლოდ:

```text
0 და 1
```

სიმბოლოებისგან და რომელშიც ერთმანეთის გვერდით ორი ერთიანი არ დგას.

მაგალითად:

```text
N = 3
```

დასაშვებია:

```text
000
001
010
100
101
```

არ დაიშვება:

```text
011
110
111
```

რადგან მათში გვერდიგვერდ ორი `1` გვხვდება.

---

# მთავარი იდეა

მოდით ვნახოთ მცირე შემთხვევები.

## N = 1

შესაძლო მიმდევრობებია:

```text
0
1
```

სულ:

```text
2
```

ამიტომ:

```cpp
dp[1] = 2;
```

---

## N = 2

შესაძლო მიმდევრობებია:

```text
00
01
10
```

სულ:

```text
3
```

რადგან

```text
11
```

აკრძალულია.

ამიტომ:

```cpp
dp[2] = 3;
```

---

## N = 3

შესაძლო მიმდევრობებია:

```text
000
001
010
100
101
```

სულ:

```text
5
```

---

აქ უკვე ფიბონაჩის მსგავსი კანონზომიერება ჩანს:

```text
2
3
5
...
```

მაგრამ რატომ?

---

# დინამიკური პროგრამირება

განვსაზღვროთ:

```text
dp[i]
```

როგორც

```text
i სიგრძის დასაშვები მიმდევრობების რაოდენობა
```

---

## როგორ მივიღოთ dp[i] ?

განვიხილოთ ნებისმიერი დასაშვები `i` სიგრძის მიმდევრობა.

მისი ბოლო სიმბოლო შეიძლება იყოს:

```text
0
```

ან

```text
1
```

---

## შემთხვევა 1: ბოლო სიმბოლო არის 0

მაგალითად:

```text
1010
0010
0000
```

თუ ბოლოს `0` დგას, მის წინ რაც უნდა იყოს.

ანუ პირველი `i-1` პოზიცია შეიძლება იყოს ნებისმიერი დასაშვები მიმდევრობა.

ასეთი რაოდენობაა:

```text
dp[i-1]
```

---

## შემთხვევა 2: ბოლო სიმბოლო არის 1

რადგან ორი ერთიანი გვერდიგვერდ არ შეიძლება იდგეს:

```text
...01
```

ბოლო ერთიანის წინ აუცილებლად უნდა იდგეს `0`.

ამიტომ ბოლო ორი სიმბოლო ფიქსირდება:

```text
01
```

---

რჩება პირველი:

```text
i-2
```

პოზიცია.

ისინი შეიძლება ნებისმიერი დასაშვები მიმდევრობა იყოს.

ასეთი რაოდენობაა:

```text
dp[i-2]
```

---

## საბოლოო ფორმულა

ორივე შემთხვევა ერთმანეთისგან დამოუკიდებელია.

ამიტომ:

```text
dp[i] = dp[i-1] + dp[i-2]
```

---

# საწყისი მნიშვნელობები

როგორც უკვე ვნახეთ:

```cpp
dp[1] = 2;
dp[2] = 3;
```

---

შემდეგ ყველა დანარჩენი მნიშვნელობა ითვლება ფორმულით:

```cpp
for(int i=3; i<=len; i++){
    dp[i]=dp[i-1]+dp[i-2];
}
```

---

# კოდის განხილვა

## მასივის შექმნა

```cpp
vector<long long> dp(len+1);
```

აქ ვინახავთ ყველა მნიშვნელობას `0`-დან `len`-მდე.

---

## საწყისი შემთხვევები

```cpp
dp[0]=0;
dp[1]=2;
dp[2]=3;
```

ამოხსნის საფუძველს სწორედ ეს მნიშვნელობები ქმნის.

---

## გარდამავალი ფორმულა

```cpp
for(int i=3; i<=len; i++){
    dp[i]=dp[i-1]+dp[i-2];
}
```

ყოველი ახალი პასუხი მიიღება წინა ორი პასუხის შეკრებით.

---

## პასუხის გამოტანა

```cpp
cout<<dp[len];
```

რადგან `dp[len]` ზუსტად აღნიშნავს `len` სიგრძის ყველა დასაშვები მიმდევრობის რაოდენობას.

---

# მაგალითი

მოცემულია:

```text
N = 5
```

გამოთვლა:

```text
dp[1] = 2
dp[2] = 3
dp[3] = 5
dp[4] = 8
dp[5] = 13
```

პასუხი:

```text
13
```

---

# რატომ მუშაობს ეს?

ნებისმიერი დასაშვები მიმდევრობა აუცილებლად სრულდება:

```text
0-ით
```

ან

```text
01-ით
```

სხვა ვარიანტი არ არსებობს, რადგან `11` აკრძალულია.

---

ამიტომ ყველა დასაშვები მიმდევრობა ზუსტად ორ ჯგუფად იყოფა:

```text
ბოლოში 0
```

და

```text
ბოლოში 01
```

პირველი ჯგუფის რაოდენობაა:

```text
dp[i-1]
```

მეორის:

```text
dp[i-2]
```

მათი ჯამი კი გვაძლევს მთლიან პასუხს.

---

# სირთულე

## დროითი სირთულე

```text
O(N)
```

რადგან `dp` მასივს მხოლოდ ერთხელ ვავსებთ.

---

## მეხსიერების სირთულე

```text
O(N)
```

რადგან ვინახავთ `N+1` ელემენტს.

---

# შეჯამება

ამოხსნა ეფუძნება დინამიკურ პროგრამირებას.

თუ `i` სიგრძის მიმდევრობის ბოლო სიმბოლო:

```text
0
```

მაშინ გვაქვს `dp[i-1]` ვარიანტი.

თუ ბოლო სიმბოლო:

```text
1
```

მაშინ მის წინ აუცილებლად `0` უნდა იდგეს და გვაქვს `dp[i-2]` ვარიანტი.

შედეგად ვიღებთ ფიბონაჩის ტიპის ფორმულას:

```text
dp[i] = dp[i-1] + dp[i-2]
```

რომლითაც სწრაფად ვითვლით პასუხს `N ≤ 40` შემთხვევაშიც.
