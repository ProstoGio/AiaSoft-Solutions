# ამოხსნის ახსნა

ამოცანაში საჭიროა დავითვალოთ რამდენი `N` სიგრძის ორობითი მიმდევრობა არსებობს, რომელიც შედგება მხოლოდ:

```text
0 და 1
```

სიმბოლოებისგან და რომელშიც სამი `1` ზედიზედ არასოდეს გვხვდება.

სხვა სიტყვებით, მიმდევრობა:

```text
111
```

არ უნდა შეიცავდეს.

---

```text
0

```

სულ:

```text
2
```

სულ:

```text
4
```

```cpp
dp[3] = 7;
```

# დინამიკური პროგრამირება

განვსაზღვროთ:

```text
dp[i]
```

როგორც

```text
i სიგრძის დასაშვები მიმდევრობების რაოდენობა
```

---

# როგორ მივიღოთ dp[i] ?

განვიხილოთ ნებისმიერი დასაშვები მიმდევრობა სიგრძით `i`.

მისი ბოლო ნაწილი აუცილებლად ერთ-ერთი შემდეგი ვარიანტია:

```text
...0
```

ან

```text
...10
```

ან

```text
...110
```

სხვა შემთხვევა ვერ იარსებებს.

---

## რატომ მხოლოდ ეს სამი?

თუ მიმდევრობა სრულდება `0`-ით:

```text
...0
```

ყველაფერი წესრიგშია.

---

თუ ბოლო სიმბოლო არის `1`, მის წინ შეიძლება იყოს:

```text
...01
```

ან

```text
...011
```

---

მაგრამ

```text
...111
```

უკვე აკრძალულია.

---

ამიტომ ყოველი დასაშვები მიმდევრობა ზუსტად ერთ-ერთ ჯგუფში ხვდება:

```text
ბოლოა 0
ბოლოა 10
ბოლოა 110
```

# შემთხვევა 1: მიმდევრობა მთავრდება 0-ით

```text

11010
```

ბოლო `0`-ს წინ ნებისმიერი დასაშვები მიმდევრობა შეიძლება იდგეს.

ასეთი რაოდენობაა:

```text
dp[i-1]
```

# საბოლოო ფორმულა

სამივე შემთხვევა ერთმანეთისგან დამოუკიდებელია.

ამიტომ:

```text
dp[i] = dp[i-1] + dp[i-2] + dp[i-3]
```

# კოდის განხილვა

## მასივის შექმნა

```cpp
vector<long long> dp(len+1);
```

აქ ვინახავთ ყველა პასუხს.

---

## საწყისი შემთხვევები

```cpp
dp[0] = 0;

if(len >= 1) dp[1] = 2;
if(len >= 2) dp[2] = 4;
if(len >= 3) dp[3] = 7;
```

---

## გარდამავალი ფორმულა

```cpp
for(int i=4; i<=len; i++){
    dp[i]=dp[i-1]+dp[i-2]+dp[i-3];
}
```

აქ ზუსტად ვიყენებთ მიღებულ რეკურენტულ დამოკიდებულებას.

---

## პასუხის გამოტანა

```cpp
cout << dp[len];
```

რადგან `dp[len]` სწორედ მოთხოვნილი რაოდენობაა.

---

```text
13
```

# რატომ მუშაობს ეს?

ნებისმიერი დასაშვები მიმდევრობა აუცილებლად მთავრდება ერთ-ერთი ფორმით:

```text
0
10
110
```

რადგან:

```text
111
```

აკრძალულია.

---

ამიტომ ყველა დასაშვები მიმდევრობა სამ დამოუკიდებელ ჯგუფად იყოფა:

```text
...0
...10
...110
```

მათი რაოდენობებია:

```text
dp[i-1]
dp[i-2]
dp[i-3]
```

ხოლო ჯამი გვაძლევს მთლიან პასუხს:

```text
dp[i] = dp[i-1] + dp[i-2] + dp[i-3]
```

---

# სირთულე

## დროითი სირთულე

```text
O(N)
```

რადგან `dp` მასივს მხოლოდ ერთხელ ვავსებთ.

---

## მეხსიერების სირთულე

```text
O(N)
```

რადგან ვინახავთ `N+1` ელემენტს.

# შეჯამება

ამოხსნის მთავარი იდეაა მიმდევრობების დაყოფა მათი დაბოლოების მიხედვით.

თუ მიმდევრობა სრულდება:

```text
0
```

მივიღებთ `dp[i-1]` ვარიანტს.

თუ სრულდება:

```text
10
```

მივიღებთ `dp[i-2]` ვარიანტს.

თუ სრულდება:

```text
110
```

მივიღებთ `dp[i-3]` ვარიანტს.

ამ სამი რაოდენობის შეკრებით ვიღებთ ფორმულას:

```text
dp[i] = dp[i-1] + dp[i-2] + dp[i-3]
```

რომელიც სწრაფად გვაძლევს პასუხს `N ≤ 32` შემთხვევისთვის.
