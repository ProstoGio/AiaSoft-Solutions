# განმარტება — 121. ციფრების ჯამი

## მიდგომა

რადგან რიცხვი 10^200-ს აღწევს, სტანდარტულ `long long`-ში ვერაფრით ჩავტევთ, ამიტომ სტრიქონად შემოყვანა ყველაზე სწორი გზაა. რეკურსიის იდეა მარტივია: ავიღოთ პირველი ციფრი, გადავაქციოთ რიცხვად და მივუმატოთ დანარჩენი სტრიქონის ციფრების ჯამს, რომელსაც იგივე ფუნქციით დავითვლით. `string::erase`-ით ნელ-ნელა ვამცირებ სტრიქონს, სანამ მხოლოდ ერთი ციფრი არ დარჩება, რაც რეკურსიის ბაზისია.

## სირთულე

| | |
|-|--|
| დრო | O(n^2) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- სტრიქონიდან ციფრის მისაღებად `x[0]-'0'` ყველაზე მარტივი და ეფექტური გზაა, რადგან ASCII ცხრილში რიცხვები თანმიმდევრობით მიყვება.
- რეკურსიულად `erase`-ის გამოყენება იწვევს O(n^2) სირთულეს, რადგან ყოველ ჯერზე სტრიქონის მეხსიერების გადაკოპირება ხდება.

## ხშირი შეცდომები

- უარყოფითი ნიშნის გათვალისწინება დაგავიწყდებათ, რაც `s[0]=='-'` შემოწმების გარეშე შეცდომას მოგცემთ.
