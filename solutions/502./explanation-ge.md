# განმარტება — 502. პიცა

## მიდგომა

მოკლედ რომ ვთქვათ, ეს არის ტიპური LCM-ის (Least Common Multiple) ამოცანა. რადგან თაკოც ჭამს პიცას, სტუმრების რაოდენობა კი N ან M შეიძლება იყოს, რეალურად გვაინტერესებს რიცხვი, რომელიც იყოფა (N+1)-ზეც და (M+1)-ზეც. C++-ის ჩაშენებულ `std::lcm` ფუნქციას თუ გამოვიყენებთ, კოდი ძალიან მარტივდება.

## სირთულე

| | |
|-|--|
| დრო | O(log(min(n, m))) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- ყველაზე მნიშვნელოვანი ისაა, რომ თაკო არ დაგავიწყდეს - ანუ (N+1) და (M+1) უნდა დაითვალო.
- LCM-ის მათემატიკური თვისებაა, რომ ის უზრუნველყოფს ნაჭრების ისეთ მინიმალურ რაოდენობას, რაც ორივე შემთხვევაში თანაბრად გადანაწილდება.

## ხშირი შეცდომები

- თაკოს დამატება თუ დაგავიწყდა, პასუხი მცდარი იქნება.
- ზოგჯერ ხალხი ცდილობს დაშლის მეთოდებით ან რაიმე რთული ციკლით ამოხსნას, როცა ერთი LCM საკმარისია.
