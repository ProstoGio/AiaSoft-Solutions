# განმარტება — 603. მონეტების ჯამი

## მიდგომა

ეს არის კლასიკური 'subset sum' ამოცანა. გადავწყვიტე DP-ით მივდგომოდი: `dp[i][j]` მიჩვენებს, შემიძლია თუ არა `j` თანხის აწყობა პირველი `i` მონეტის გამოყენებით. ყოველ ახალ მონეტაზე ფაქტობრივად ორი გზა გვაქვს: ან არ ვიყენებთ ამ მონეტას (ვიტოვებთ წინა ჯამებს), ან ვუმატებთ მას უკვე მიღწევად ყველა ჯამს. საკმაოდ პირდაპირი ლოგიკაა, მთავარია სწორად გავატაროთ მდგომარეობები.

## სირთულე

| | |
|-|--|
| დრო | O(n * Σx) |
| მეხსიერება | O(n * Σx) |

## მთავარი დაკვირვებები

- მთელი სუბსეტების შენახვა არ გვჭირდება, მარტო ის 'ფლაგები' გვყოფნის, რაც გვეუბნება - მიღწევადია თუ არა ჯამი.
- მიუხედავად იმისა, რომ 2D მასივი გამოვიყენე, მეხსიერების დაზოგვა 1D მასივითაც შეიძლებოდა, თუ ციკლს უკნიდან წამოვიყვანდით.

## ხშირი შეცდომები

- ყველაზე ხშირი შეცდომაა თავად მონეტის მნიშვნელობის დავიწყება, რომელიც ცალკე ჯამია.
- მაქსიმალური ჯამის ზღვრის არასწორად გათვლა - უნდა გვახსოვდეს, რომ n*max(xi) არის ზედა ზღვარი.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ მონეტებია {2, 3}. თავიდან dp[0][2]=1. შემდეგ, როცა 3-ს განვიხილავთ: ჯერ 3-ს ვნიშნავთ როგორც მიღწევადს, მერე ვუმატებთ მას წინა ნაპოვნ 2-ს და ვიღებთ 5-ს. შედეგად გვაქვს {2, 3, 5}.
