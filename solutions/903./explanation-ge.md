# განმარტება — 903. ქალაქ ბაიას მერი

## მიდგომა

სინამდვილეში ეს უბრალო გრაფის ამოცანაა, სადაც თითოეული მოედანი წვეროა, ხოლო ქუჩა — წიბო. რადგან N-ი მხოლოდ 100-მდეა, პირდაპირ ადიჯესენთ მატრიცას გამოვიყენებ, სადაც 1-ით აღვნიშნავ ქუჩის არსებობას. თითოეულ მოედანზე ქუჩების რაოდენობის დასათვლელად უბრალოდ მატრიცის ხაზებს ვუვლი და ვამატებ 1-ებს. ბოლოს, როცა მაქსიმალურს ვეძებ, პირობა ისეთია, რომ თანაბარი ქუჩების შემთხვევაში უფრო დიდი ინდექსი უნდა ამოვირჩიო — ამიტომაცაა `i >= mx` შედარება, რაც ავტომატურად გადააგდებს უფრო პატარა ინდექსს.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n²) |

## მთავარი დაკვირვებები

- N=100 არის საკმარისად მცირე, რომ O(n²) მატრიცა მარტივად ჩაეტიოს მეხსიერებაში.
- თუ ქუჩების რაოდენობა ტოლია, ინდექსის განახლება უდიდესი ნომრით ხდება უბრალოდ `>=` შედარების გამოყენებით.

## ხშირი შეცდომები

- ყველაზე ხშირი შეცდომაა მხოლოდ 'მეტია' შედარების გაკეთება (>) და ტოლობის შემთხვევაში დიდი ინდექსის გათვალისწინების დავიწყება.
