# განმარტება — 548. ოთახების დათვლა

## მიდგომა

ეს ამოცანა კლასიკური 'connected components'-ის პოვნაა გრაფებზე, ოღონდ გრიდის ფორმატში. როგორც კი წერტილს დავინახავ, რომელიც ჯერ არ მინახავს, ვიცი, რომ ახალი ოთახი ვიპოვე. მერე ვუშვებ DFS-ს, რომელიც ყველა მეზობელ წერტილს 'მონახულებულად' მონიშნავს. ასე მარტივად, ერთი ოთახის ყველა უჯრა ერთიანად 'ჩაიხურება' და მეორედ აღარ დავითვლი.

## სირთულე

| | |
|-|--|
| დრო | O(n * m) |
| მეხსიერება | O(n * m) |

## მთავარი დაკვირვებები

- მთავარია used მასივი, რომ რეკურსიამ ციკლში არ ჩაგდოს.
- საზღვრების შემოწმება არ დაგავიწყდეს, თორემ C++ მწარედ დაგისჯის out of bounds წვდომისთვის.

## ხშირი შეცდომები

- used მასივის განულება ან არასწორად შემოწმება.
- i და j ინდექსების არევა ან საზღვრების არასწორად დაწერა (მაგალითად i+1-ის დროს i < n-ის მაგივრად i <= n-ის დაწერა).

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ გვაქვს მატრიცა. მივყვები ციკლით თითოეულ უჯრას. როგორც კი წერტილს შევეხები და used[i][j] ნოლია, ვზრდი cnt-ს და ვიძახებ DFS-ს. DFS-ი ივლის გარშემო მანამ, სანამ 'კედლებს' ან უკვე მონახულებულ უჯრებს არ შეეჯახება.
