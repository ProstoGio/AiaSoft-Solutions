# განმარტება — 350. იძებნება თუ არა?

## მიდგომა

სინამდვილეში ამ ამოცანის ამოხსნა მარტივი 'for' ციკლითაც შეიძლებოდა, მაგრამ ბინარული ძებნა უფრო 'პროფესიონალური' და ეფექტური გზაა, თან სასწავლოდაც კარგია. თავიდან მასივს ვაწყობთ ზრდადობით, რაც აუცილებელია, რომ ბინარულმა ძებნამ იმუშაოს. მერე უკვე ორად ვყოფთ შუალედს და ვამოწმებთ, სად შეიძლება იყოს ჩვენი რიცხვი, სანამ საძიებო არეალი ერთ წერტილამდე არ შემცირდება.

## სირთულე

| | |
|-|--|
| დრო | O(n log n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- Sort-ის გარეშე ბინარული ძებნა არ იმუშავებს, ასე რომ ჯერ სორტირებაა მთავარი.
- მთლიანი მასივის გადათვალიერების ნაცვლად, ყოველ ჯერზე შუაზე ვყოფთ სეგმენტს, რაც დროში გვზოგავს.

## ხშირი შეცდომები

- სორტირების გამორჩენა, რის გამოც ბინარული ძებნა სისულელეს ამოაგდებს.
