# განმარტება — 257. რიცხვის ძიება მასივში - 2

## მიდგომა

პირველი მასივი დალაგებულია და ეს ყველაზე დიდი მინიშნებაა, რომ `linear search`-ს არ უნდა გავეკაროთ. თითოეული რიცხვისთვის `M` მასივიდან, `binary search`-ით ვამოწმებ `a` მასივში არსებობას, რაც საგრძნობლად ამცირებს დროის დანახარჯს. პრინციპი მარტივია: შუა ელემენტს ვედარები და ნახევარ მასივს ყოველ ჯერზე ვთიშავ საძიებო არეალიდან.

## სირთულე

| | |
|-|--|
| დრო | O(M log N) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- დალაგებული მასივი ყოველთვის გვეუბნება, რომ 'divide and conquer' სტრატეგია უნდა ავირჩიოთ.
- O(M log N) სრულიად საკმარისია 10^6 ელემენტიანი მასივისთვის, მაშინ როცა brute force ნამდვილად გაიჭედებოდა.

## ხშირი შეცდომები

- ნელი ძებნის მეთოდის გამოყენება. თუ `n` მილიონია და `m` ათასი, `O(n*m)` მიდგომა 1.0 წამში ვერ ჩაეტევა.

## ნაბიჯ-ნაბიჯ ახსნა

ავიღოთ `l=0` და `r=n-1`. ვიანგარიშოთ `mid = (l+r)/2`. თუ `a[mid]` მეტია ჩვენს რიცხვზე, `r`-ს ვწევთ მარცხნივ, თუ ნაკლებია - `l`-ს ვწევთ მარჯვნივ. თუ დავამთხვიეთ, `ind=1` და ვწყვეტთ ციკლს.
