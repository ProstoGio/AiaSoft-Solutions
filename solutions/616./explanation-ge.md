# განმარტება — 616. ბიტებით თამაში

## მიდგომა

სინამდვილეში აქ რთული არაფერია, უბრალოდ უნდა იცოდე რომელი ბიტური ოპერაცია რას აკეთებს. როცა bit-ის ჩართვა გინდა, OR-ს იყენებ, ჩასაქრობად AND-ს, ხოლო შესაცვლელად - XOR-ს. მთავარია სწორად ააწყო 'mask'-ი `1LL << k`-ით. მარტივი ლოგიკაა: თუ იცი, როგორ მუშაობს ეს ოპერატორები, პრობლემა წამებში იხურება.

## სირთულე

| | |
|-|--|
| დრო | O(Q) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- ყოველთვის დაწერე 1LL, თორემ C++ 1-ს ჩვეულებრივ int-ად აღიქვამს და დიდი k-ებისთვის სერიოზულ პრობლემებს აგკიდებს.
- XOR ოპერატორი t=3-ისთვის ზედგამოჭრილია, რადგან 1-ზე XOR-ი ყოველთვის ცვლის მნიშვნელობას საპირისპიროზე.

## ხშირი შეცდომები

- ინტეგერების ლიმიტების დავიწყება. თუ 1-ს დატოვებ `1LL`-ის გარეშე, ბევრი ბიტი დაიკარგება shift-ის დროს.
