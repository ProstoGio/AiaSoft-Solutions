# ამოხსნის ახსნა

მოცემულია ერთი რიცხვი `N`.

საჭიროა გამოვთვალოთ:

```text
1 ⊕ 2 ⊕ 3 ⊕ ... ⊕ N
```

სადაც `⊕` აღნიშნავს ბიტურ **XOR** ოპერაციას.

ამ ამოცანაში `N` შეიძლება იყოს ძალიან დიდი:

```text
1 ≤ N ≤ 10¹⁸
```

ამიტომ ყველა რიცხვზე გავლა შეუძლებელია.

---

# რატომ არ გამოდგება ჩვეულებრივი ციკლი?

პატარა `N`-ისთვის შეგვეძლო ასე გვექნა:

```cpp
for(int i=1; i<=N; i++)
    answer ^= i;
```

მაგრამ თუ:

```text
N = 10¹⁸
```

ციკლი ტრილიონობით ნაბიჯს შეასრულებს, რაც პრაქტიკულად შეუძლებელია.

საჭიროა უფრო სწრაფი მეთოდი.

---

# მთავარი დაკვირვება

თუ პატარა მნიშვნელობებისთვის ხელით დავითვლით XOR-ს, მივიღებთ:

| N | 1⊕2⊕...⊕N |
|---|-----------|
| 1 | 1 |
| 2 | 3 |
| 3 | 0 |
| 4 | 4 |
| 5 | 1 |
| 6 | 7 |
| 7 | 0 |
| 8 | 8 |

---

თუ დავაკვირდებით, პასუხი ყოველ ოთხ რიცხვში ერთხელ მეორდება.

ანუ ყველაფერი დამოკიდებულია მხოლოდ იმაზე, თუ რა ნაშთს იძლევა `N` ოთხზე გაყოფისას.

---

# კოდის განხილვა

## რიცხვის წაკითხვა

```cpp
long long n;
cin >> n;
```

---

## პირველი შემთხვევა

```cpp
if(n % 4 == 0)
    cout << n;
```

---

## მეორე შემთხვევა

```cpp
else if(n % 4 == 1)
    cout << 1;
```

---

## მესამე შემთხვევა

```cpp
else if(n % 4 == 2)
    cout << n + 1;
```

---

## მეოთხე შემთხვევა

```cpp
else if(n % 4 == 3)
    cout << 0;
```

# რატომ მუშაობს?

რიცხვების XOR-ს ოთხი მნიშვნელობის ციკლი აქვს.

ყოველთვის, როცა `N` ოთხით იზრდება, პასუხის ფორმულაც იგივე წესით მეორდება.

ამიტომ საკმარისია მხოლოდ `N % 4`-ის მნიშვნელობის შემოწმება.

ამის შემდეგ უკვე პირდაპირ ვიცით სწორი პასუხი და აღარ გვჭირდება ყველა რიცხვის XOR-ის გამოთვლა.

# შეჯამება

რადგან `1`-დან `N`-მდე რიცხვების XOR ყოველ ოთხ რიცხვში ერთხელ იმეორებს ერთსა და იმავე კანონზომიერებას, საკმარისია ვნახოთ `N`-ის ნაშთი `4`-ზე გაყოფისას.

ამის მიხედვით პირდაპირ ვბეჭდავთ პასუხს, რაც საშუალებას გვაძლევს ამოვხსნათ ამოცანა მუდმივ დროში, მაშინაც კი, როცა `N` ძალიან დიდია.
