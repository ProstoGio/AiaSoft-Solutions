# განმარტება — 264. კომპონენტების რაოდენობა

## მიდგომა

გრაფის კომპონენტების დასათვლელად ყველაზე მარტივი გზა DSU-ს (Disjoint Set Union) გამოყენებაა. თავიდან ყველა წვერო ცალკე კომპონენტად მიგვაჩნია, ანუ თითოეული თავისი თავის 'მამაა'. როგორც კი წიბოს შემოვიკითხავთ, ორივე წვეროს ერთ კომპონენტში ვაერთიანებთ `union_set`-ით. ბოლოს უბრალოდ იმ წვეროებს ვითვლით, რომლებიც ჯერ კიდევ საკუთარი თავის 'მამები' არიან – თითოეული ასეთი წვერო თითო კომპონენტს აღნიშნავს.

## სირთულე

| | |
|-|--|
| დრო | O(M * α(N)) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- DSU-ს გამოყენება ბევრად უფრო კომპაქტურია, ვიდრე თითოეული წვეროდან DFS-ის ან BFS-ის გაშვება.
- როცა parent[i] == i, ეს იმას ნიშნავს, რომ იპოვე კომპონენტის 'ფესვი', რაც ავტომატურად ზრდის კომპონენტების რაოდენობას.

## ხშირი შეცდომები

- პარენთების ინიციალიზაციის გამორჩენა (თავიდან `parent[i] = i` უნდა იყოს, თორემ ყველაფერი აირევა).
