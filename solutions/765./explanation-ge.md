# განმარტება — 765. დიაპაზონი

## მიდგომა

რადგან რიცხვები 100 000-ს არ ცდება, არ გვაქვს იმის საჭიროება, რომ რთული სტრუქტურები, მაგალითად Segment Tree ავაწყოთ. ყველაზე ეფექტური აქ frequency array-ის გამოყენებაა. ჯერ ვითვლით თითოეული რიცხვი რამდენჯერ შეგვხვდა, მერე ვაკეთებთ prefix sum-ს და ნებისმიერი დიაპაზონისთვის პასუხს O(1)-ში ვიღებთ.

## სირთულე

| | |
|-|--|
| დრო | O(N + M + max(V)) |
| მეხსიერება | O(max(V)) |

## მთავარი დაკვირვებები

- რიცხვების შეზღუდული დიაპაზონი პირდაპირ გვეუბნება, რომ 'counting sort'-ის სტილის მასივი იდეალურია.
- Prefix sum-ით დიაპაზონების ჯამების გამოთვლა კლასიკური ტრიუკია, რომელიც აქაც ზუსტად მაგრად მუშაობს.

## ხშირი შეცდომები

- თუ A ან B რიცხვი უფრო დიდია, ვიდრე მიმდევრობაში არსებული მაქსიმუმი, ეგ შემთხვევა ცალკე უნდა გაითვალისწინო, თორემ კოდი 'Runtime Error'-ს ააგდებს.
- ინდექსებთან თამაში: prefix sum-ში ზუსტად უნდა მიხვდე სად იწყება და სად მთავრდება დიაპაზონი, რომ 'off-by-one' არ გაგეპაროს.

## ნაბიჯ-ნაბიჯ ახსნა

წარმოიდგინე, გაქვს მასივი [1, 2, 2, 5]. frequency მასივი იქნება [0, 1, 2, 0, 0, 1]. ამის prefix sum-ს რომ ააგებ, [A, B] დიაპაზონისთვის უბრალოდ წაიკითხავ s2[B] - s2[A] + s[A], რაც მომენტალურად გაძლევს პასუხს.
