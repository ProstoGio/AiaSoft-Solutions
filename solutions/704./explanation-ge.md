# განმარტება — 704. ინტერვალის ჯამი

## მიდგომა

პირდაპირ რომ დავუაროთ მასივს ყოველი ქველისთვის, მინიმუმ O(N*Q) გამოვა და 2 წამი ნამდვილად არ გვეყოფა, განსაკუთრებით 2*10^5-ზე. ამიტომ ვაკეთებთ `dp` მასივს, სადაც ვინახავთ წინა ჯამებს: `dp[i]` არის მასივის დასაწყისიდან `i`-მდე ელემენტების ჯამი. მერე უკვე [l, r] ინტერვალის საპოვნელად უბრალოდ `dp[r-1] - dp[l-2]`-ს ვაკეთებთ. ეს არის ყველაზე კლასიკური ტრიკი, რომ query-ები O(1)-ში დავხუროთ.

## სირთულე

| | |
|-|--|
| დრო | O(N + Q) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- Prefix sum-ით ნებისმიერი ინტერვალის ჯამი ხდება მარტივი გამოკლება.
- აუცილებლად long long-ი გამოიყენე, თორემ 10^9-ები რომ იკრიბება, integer-ის ლიმიტს მარტივად ცდება.

## ხშირი შეცდომები

- 1-დან დაწყებული ინდექსების გამო არასწორი ინდექსით მიმართვა (off-by-one errors).
- int-ის გამოყენება ჯამებისთვის, რაც overflow-ს იწვევს.
