# განმარტება — 320. ტალღა მატრიცაში

## მიდგომა

ეს არის კლასიკური BFS-ი, უბრალოდ ვიზუალურად ტალღას ჰგავს. რადგან თითოეულ უჯრაზე გადასვლა ერთი ნაბიჯია, რიგის (queue) გამოყენება იდეალურია, რადგან ყოველთვის ზუსტად იმ თანმიმდევრობით დაამუშავებს უჯრებს, რა მანძილზეც არიან საწყისი წერტილიდან. პრინციპი მარტივია: საწყისი წერტილიდან ვიწყებთ, ვამოწმებთ მეზობლებს და ვანიჭებთ მანძილს, თუ ადრე არ გვაქვს ნანახი.

## სირთულე

| | |
|-|--|
| დრო | O(N*M) |
| მეხსიერება | O(N*M) |

## მთავარი დაკვირვებები

- BFS-ი ავტომატურად ინახავს ტალღის პრინციპს, რადგან ის ჯერ უახლოეს უჯრებს ავსებს და მერე გადადის უფრო შორს.
- used მასივი აუცილებელია, რომ უსასრულო ციკლში არ გავეხვიოთ და ერთი და იგივე უჯრა რამდენჯერმე არ დავამუშაოთ.

## ხშირი შეცდომები

- საზღვრების შემოწმების გამორჩენა (if condition-ები), რამაც შეიძლება memory-ს დაზიანება ან Runtime Error გამოიწვიოს.
- რიგში ჩაყრისას used მასივის დროულად არ მონიშვნა.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ, ვიწყებთ (2, 2)-დან. ვდებთ queue-ში, ვანიჭებთ 0-ს. მერე ამოვიღებთ და ვუყურებთ მის მეზობლებს. ვინც ჯერ არაა მონახულებული, ყველას ვანიჭებთ 0+1=1-ს და ვდებთ რიგში. შემდეგ ეტაპზე რიგიდან ამოვა '1'-ები, მათ მეზობლებს მივანიჭებთ '2'-ს და ასე გავავსებთ მთელ მატრიცას.
