# ამოხსნის ახსნა

მოცემულია არაუარყოფითი მთელი რიცხვი `X` და რიცხვი `i`.

საჭიროა გავიგოთ, რა მნიშვნელობა აქვს `X`-ის ორობით ჩანაწერში `i`-ე ბიტს.

ბიტების ათვლა იწყება მარჯვნიდან.

---

# მაგალითი

თუ:

```text
X = 13
i = 2
```

რიცხვი `13` ორობით სისტემაში ჩაიწერება ასე:

```text
1101
```

მარჯვნიდან ბიტების ნომრებია:

```text
1101
3210
```

`2`-ე პოზიციაზე დგას:

```text
1
```

ამიტომ პასუხია:

```text
1
```

---

კიდევ ერთი მაგალითი.

თუ:

```text
X = 10
i = 0
```

ორობით ჩანაწერია:

```text
1010
```

მარჯვნიდან:

```text
1010
3210
```

`0`-ე ბიტია:

```text
0
```

---

# მთავარი იდეა

საჭიროა მხოლოდ ერთი კონკრეტული ბიტის შემოწმება.

ამისათვის ვიყენებთ ორ ბიტურ ოპერაციას:

- მარჯვნივ გადაწევას (`>>`);
- ბიტურ AND ოპერაციას (`&`).

---

# პირველი ნაბიჯი

რიცხვს `i` ბიტით მარჯვნივ ვწევთ.

```cpp
x >> k
```

ამის შემდეგ ჩვენთვის საინტერესო ბიტი ყველაზე მარჯვენა პოზიციაზე აღმოჩნდება.

---

მაგალითად:

```text
X = 13

1101
```

თუ:

```text
i = 2
```

მაშინ:

```text
1101 >> 2
```

მივიღებთ:

```text
11
```

ახლა საჭირო ბიტი უკვე ყველაზე მარჯვნივ დგას.

---

# მეორე ნაბიჯი

ვაკეთებთ:

```cpp
& 1
```

რიცხვი `1` ორობითად ასე გამოიყურება:

```text
0001
```

---

AND ოპერაცია მხოლოდ ბოლო ბიტს ტოვებს.

მაგალითად:

```text
0011
0001
----
0001
```

შედეგია:

```text
1
```

---

თუ ბოლო ბიტი იყო `0`:

```text
0010
0001
----
0000
```

შედეგი იქნება:

```text
0
```

---

# ყველაფერი ერთად

კოდში ეს ორი მოქმედება გაერთიანებულია:

```cpp
((x >> k) & 1)
```

---

ჯერ ხდება გადაწევა:

```text
x >> k
```

შემდეგ კი:

```text
& 1
```

და შედეგად ვიღებთ მხოლოდ იმ ბიტის მნიშვნელობას, რომელიც გვაინტერესებდა.

---

# კოდის განხილვა

## მონაცემების წაკითხვა

```cpp
long long x, k;
cin >> x >> k;
```

---

## ბიტის მიღება

```cpp
cout << ((x >> k) & 1);
```

აქ:

- `>>` რიცხვს მარჯვნივ წევს;
- `& 1` მხოლოდ ბოლო ბიტს ტოვებს.

სწორედ ეს ბოლო ბიტი არის `i`-ე პოზიციაზე არსებული მნიშვნელობა.

---

# რატომ მუშაობს?

რიცხვის მარჯვნივ გადაწევისას ყველა ბიტი `i` პოზიციით ინაცვლებს.

შედეგად, `i`-ე ბიტი ყველაზე მარჯვენა ადგილას აღმოჩნდება.

ამის შემდეგ `& 1` მხოლოდ ამ ბიტს ტოვებს და ყველა დანარჩენს შლის.

ამიტომ საბოლოოდ ვიღებთ ზუსტად `0`-ს ან `1`-ს, რაც მოთხოვნილი ბიტის მნიშვნელობაა.

---

# სირთულე

### დროითი სირთულე

```text
O(1)
```

სრულდება მხოლოდ რამდენიმე ბიტური ოპერაცია.

---

### მეხსიერების სირთულე

```text
O(1)
```

დამატებითი მეხსიერება არ გამოიყენება.

---

# შეჯამება

ამ ამოცანის მთავარი იდეაა, რომ ჯერ საჭირო ბიტი მარჯვენა ბოლოს გადმოვიტანოთ `>>` ოპერაციით, შემდეგ კი `& 1` ოპერაციით მხოლოდ ეს ბიტი დავტოვოთ.

ასე ძალიან სწრაფად ვიგებთ, არის თუ არა მოცემულ პოზიციაზე `0` თუ `1`.
