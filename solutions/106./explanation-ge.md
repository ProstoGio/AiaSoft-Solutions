# განმარტება — 106. ლიდერები

## მიდგომა

თავიდან რომ დავიწყო მარცხნიდან მარჯვნივ ყოველი ელემენტისთვის იმის შემოწმება, მის მარჯვნივ რა ხდება, O(n²) გამოვა, რაც ამხელა n-ისთვის უეჭველი TLE-ია. მარტივი ხრიკია უკნიდან წამოსვლა. ბოლო ელემენტი ხომ ლიდერია, ამიტომ მას ვიმახსოვრებთ როგორც 'მიმდინარე მაქსიმუმს'. მარცხნივ მოძრაობისას, როგორც კი რამე უფრო დიდი ან ტოლი გვხვდება, ეგეც ლიდერია და ეგ ხდება ახალი მაქსიმუმი. ბოლოს უბრალოდ ამოტრიალებულად გამოგვაქვს, რადგან უკნიდან წამოსულებმა ლიდერები 'შებრუნებულ' წესრიგში დავაგროვეთ.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- უკნიდან წინ სკანირება გვაძლევს საშუალებას, რომ 1-ელ პასში ამოვხსნათ ამოცანა.
- მხოლოდ მაქსიმალური ელემენტის შენახვაა საკმარისი იმის დასადგენად, არის თუ არა რიცხვი ლიდერი.

## ხშირი შეცდომები

- O(n²) მიდგომის არჩევა, როცა n=10⁶-ია.
