# განმარტება — 604. შეკითხვები ტყეზე

## მიდგომა

საკითხი მარტივია: თუ ყველა ქუერიზე ნულიდან თავიდან ვიანგარიშებთ ხეების რაოდენობას, TLE-ს ავიკიდებთ. ამიტომ ვიყენებთ 2D prefix sums. წინასწარ ვითვლით მატრიცას, სადაც თითოეული უჯრა ინახავს ხეების ჯამს (0,0)-დან ამ წერტილამდე. ამის მერე, ნებისმიერი მართკუთხედის ფართობი პასუხდება O(1)-ში, inclusion-exclusion-ის პრინციპით.

## სირთულე

| | |
|-|--|
| დრო | O(N² + Q) |
| მეხსიერება | O(N²) |

## მთავარი დაკვირვებები

- ჯამის დასათვლელად ფორმულაა: მიმდინარე უჯრა + ზედა უჯრა + მარცხენა უჯრა - დიაგონალური (რომელიც ორჯერ დაემატა).
- როცა მართკუთხედის ფართობს ვთვლით, მთავარია სწორად გამოვაკლოთ ზედა და მარცხენა არეები და დავუმატოთ ის პატარა ნაწილი, რომელიც ზედმეტად გამოვაკელით.

## ხშირი შეცდომები

- ყველაზე ხშირი შეცდომაა 1-based ინდექსების 0-based-ში არასწორი გადაყვანა, რის გამოც მატრიცის საზღვრებს ცდებიან.

## ნაბიჯ-ნაბიჯ ახსნა

მაგალითად, თუ გაინტერესებს [x0, y0]-დან [x1, y1]-მდე ხეები, იღებ d[x1][y1]-ს, აკლებ d[x1][y0-1] და d[x0-1][y1]-ს, და ბოლოს უმატებ d[x0-1][y0-1]-ს, რადგან დიაგონალი ორივე გამოკლებაში შევიდა.
