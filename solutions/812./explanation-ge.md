# განმარტება — 812. უდიდესი კენტი

## მიდგომა

საკმაოდ მარტივი ამოცანაა, მთავარია კენტი რიცხვები გამოვარჩიოთ. ცვლადს, სადაც მაქსიმალურ მნიშვნელობას ვინახავთ, თავიდანვე 0-ს ვუტოლებ. ციკლში თითოეულ ელემენტს ვამოწმებ: თუ კენტია და თან ჩემს შენახულ `k`-ზე დიდია, მაშინ `k` ხდება ეს ახალი რიცხვი. ბოლოს, თუ `k` ისევ 0 დარჩა, ესე იგი კენტი რიცხვი საერთოდ არ შეგვხვედრია და -1-ს ვბეჭდავთ.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- 0-ით ინიციალიზება იმიტომ გამოდის კარგად, რომ ამოცანაში ნატურალური რიცხვებია და კენტი რიცხვი 0 ვერ იქნება.
- გვირჩევნია ერთი `if`-ით შევამოწმოთ კენტიც და მაქსიმუმის პირობაც, კოდი უფრო სუფთა გამოდის.

## ხშირი შეცდომები

- თუ შემთხვევით მაქსიმუმის საწყის მნიშვნელობად 1-ს ჩასვამთ, შეიძლება ლოგიკა აგერიოთ, როცა მასივში მხოლოდ 1-ზე ნაკლები კენტი რიცხვებია (თუმცა აქ ნატურალური რიცხვებია, ასე რომ 0 ყველაზე უსაფრთხოა).
