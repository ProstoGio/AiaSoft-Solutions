# განმარტება — 940. მარტივი ციკლები

## მიდგომა

თავიდან რომ შევხედე, მეგონა DFS-ით ან რაიმე რთული ალგორითმით მომიწევდა ციკლების ძებნა. მერე მივხვდი, რომ რეალურად გვეკითხებიან რამდენი ზედმეტი წიბო გვაქვს, რომელიც უკვე შეერთებულ წვეროებს აკავშირებს. როგორც კი დავინახავ, რომ edge-ის ორივე ბოლო უკვე ერთ set-შია, ეგრევე ვიცი, რომ ციკლი შეიკრა. DSU ამისთვის იდეალურია, იმიტომ რომ უბრალოდ კომპონენტებს ვამოწმებთ და ზედმეტი თავის ტკივილი არ გვჭირდება.

## სირთულე

| | |
|-|--|
| დრო | O(M * α(N)) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- თუ edge-ის თავები უკვე ერთ set-შია, ესე იგი ეგ წიბო ციკლს კრავს.
- DSU-ს გამოყენებით პირდაპირ ვითვლით რამდენი ასეთი 'დამატებითი' წიბო გვაქვს.

## ხშირი შეცდომები

- გრაფის სტრუქტურის ზედმეტად რთულად აღწერა, როცა მხოლოდ კავშირი გვაინტერესებს.
