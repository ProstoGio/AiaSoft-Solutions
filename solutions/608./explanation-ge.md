# განმარტება — 608. რიცხვების XOR - 2

## მიდგომა

N 10^18-მდე ადის, ასე რომ ჩვეულებრივი loop-ით სიარული აზრი არ აქვს, Time Limit-ში ვერ ჩაეტევი. თუ რამდენიმე პატარა რიცხვის XOR-ს დაწერ ფურცელზე, დაინახავ, რომ პასუხი 4-იანი ციკლით მეორდება. მაგალითად, 1-დან 4-მდე რიცხვებს თუ შეკრიბავ XOR-ით, მიიღებ 4-ს, მერე 1-ს, N+1-ს და 0-ს. ამის მერე კოდი სულ რამდენიმე if-ია.

## სირთულე

| | |
|-|--|
| დრო | O(1) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- XOR-ის შედეგები 4-ზე გაყოფის ნაშთზეა დამოკიდებული: n%4-ის მიხედვით ზუსტად იცი პასუხი რა იქნება.
- თუ ნაშთი 0-ია პასუხი N-ია, თუ 1-ია პასუხი 1-ია, თუ 2-ია N+1, ხოლო თუ 3-ია მაშინ 0-ს გვიბრუნებს.

## ხშირი შეცდომები

- O(N) მიდგომის არჩევა, რაც ასეთი დიდი N-ისთვის არ იმუშავებს.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ N=6. 6%4 არის 2, ანუ პასუხი უნდა იყოს N+1, ანუ 7. მართლაც: 1^2=3, 3^3=0, 0^4=4, 4^5=1, 1^6=7. ემთხვევა!
