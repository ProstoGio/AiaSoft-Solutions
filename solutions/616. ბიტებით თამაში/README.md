# ამოხსნის ახსნა

მოცემულია `Q` მოთხოვნა.

თითოეული მოთხოვნა შეიცავს სამ რიცხვს:

```text
x, k, t
```

სადაც:

- `x` — მოცემული რიცხვია;
- `k` — ბიტის პოზიცია (ათვლა იწყება მარჯვნიდან);
- `t` — მოთხოვნის ტიპი.

---

# მოთხოვნების ტიპები

## თუ

```text
t = 1
```

უნდა **ჩავრთოთ** `k`-ე ბიტი.

ანუ ამ ბიტს მნიშვნელობა უნდა ჰქონდეს `1`, მიუხედავად იმისა, ადრე `0` იყო თუ `1`.

---

## თუ

```text
t = 2
```

უნდა **გამოვრთოთ** `k`-ე ბიტი.

ანუ ამ ბიტს მნიშვნელობა უნდა ჰქონდეს `0`.

---

## თუ

```text
t = 3
```

უნდა **შევცვალოთ** `k`-ე ბიტი.

თუ `0` იყო, გახდება `1`.

თუ `1` იყო, გახდება `0`.

---

# მთავარი იდეა

სამივე მოქმედება ძალიან მარტივად სრულდება ბიტური ოპერაციებით.

გამოვიყენებთ:

- `|` (OR)
- `&` (AND)
- `~` (NOT)
- `^` (XOR)
- `<<` (მარცხნივ გადაწევა)

---

# ბიტის ჩართვა

```cpp
x | (1LL << k)
```

---

## რას აკეთებს?

ჯერ ვქმნით რიცხვს, რომელსაც მხოლოდ `k`-ე ბიტი აქვს ჩართული.

მაგალითად:

```text
k = 3
```

მაშინ:

```text
1 << 3

0001
↓

1000
```

შემდეგ ვაკეთებთ OR ოპერაციას.

OR-ის თვისებაა:

```text
0 | 1 = 1
1 | 1 = 1
```

ანუ `k`-ე ბიტი აუცილებლად გახდება `1`.

---

# ბიტის გამორთვა

```cpp
x & (~(1LL << k))
```

---

ჯერ ისევ ვქმნით მასკას:

```text
1 << k
```

მაგრამ შემდეგ ვიყენებთ `~` ოპერაციას.

მაგალითად:

```text
00010000
```

გახდება:

```text
11101111
```

ახლა AND ოპერაცია ყველა ბიტს უცვლელად ტოვებს, გარდა `k`-ე ბიტისა, რომელიც აუცილებლად `0` გახდება.

---

# ბიტის შეცვლა

```cpp
x ^ (1LL << k)
```

XOR ოპერაციის მთავარი თვისებაა:

```text
0 ^ 1 = 1
1 ^ 1 = 0
```

ანუ მხოლოდ მითითებული ბიტი შეიცვლება.

თუ ჩართული იყო, გამოირთვება.

თუ გამორთული იყო, ჩაირთვება.

---

# კოდის განხილვა

## მოთხოვნების რაოდენობის წაკითხვა

```cpp
int n;
cin >> n;
```

---

## ყველა მოთხოვნის დამუშავება

```cpp
while(n--){
```

ციკლი სრულდება `Q`-ჯერ.

ყოველ ჯერზე იკითხება ახალი მოთხოვნა.

---

## მონაცემების წაკითხვა

```cpp
long long x, k, t;
cin >> x >> k >> t;
```

---

## პირველი ტიპი

```cpp
if(t==1){
    cout << (x | (1LL << k)) << "\n";
}
```

OR ოპერაციით ვრთავთ `k`-ე ბიტს.

---

## მეორე ტიპი

```cpp
else if(t==2){
    cout << (x & (~(1LL << k))) << "\n";
}
```

AND და NOT ოპერაციების დახმარებით ვთიშავთ `k`-ე ბიტს.

---

## მესამე ტიპი

```cpp
else if(t==3){
    cout << (x ^ (1LL << k)) << "\n";
}
```

XOR ოპერაციით ვცვლით `k`-ე ბიტს.

---

# რატომ გამოიყენება `1LL`?

კოდში წერია:

```cpp
1LL << k
```

და არა უბრალოდ:

```cpp
1 << k
```

`1LL` არის `long long` ტიპის რიცხვი.

ამის გამო გადაწევაც `long long` ტიპში სრულდება, რაც უფრო უსაფრთხოა დიდი რიცხვებისთვის და თავიდან გვარიდებს შესაძლო გადავსებას (overflow).

---

# რატომ მუშაობს?

თითოეული ბიტური ოპერაცია ზუსტად იმ მოქმედებისთვისაა შექმნილი, რომელიც ამოცანაში გვჭირდება.

- `|` ყოველთვის რთავს ბიტს;
- `&` და `~` ერთად ყოველთვის თიშავენ ბიტს;
- `^` ყოველთვის ცვლის ბიტის მნიშვნელობას.

ამიტომ თითოეული მოთხოვნა სრულდება მხოლოდ ერთი ბიტური გამოსახულებით.

---

# სირთულე

### დროითი სირთულე

```text
O(Q)
```

თითოეული მოთხოვნა მუშავდება მუდმივ დროში `O(1)`.

---

### მეხსიერების სირთულე

```text
O(1)
```

დამატებითი მეხსიერება პრაქტიკულად არ გამოიყენება.

---

# შეჯამება

ამ ამოცანის მთავარი იდეაა ბიტური ოპერაციების გამოყენება.

- `OR (|)` გამოიყენება ბიტის ჩასართავად.
- `AND (&)` და `NOT (~)` გამოიყენება ბიტის გამოსართავად.
- `XOR (^)` გამოიყენება ბიტის შესაცვლელად.

ამის წყალობით თითოეული მოთხოვნა სრულდება ძალიან სწრაფად, მხოლოდ რამდენიმე ბიტური ოპერაციით.
