# ამოხსნის ახსნა

ამოცანაში მოცემულია რიცხვების გადანაცვლება.

თამაშის წესები:

- ტომი იწყებს თამაშს.
- თითო სვლაზე მოთამაშეს შეუძლია მიმდევრობიდან წაშალოს ზუსტად ერთი ელემენტი.
- თამაში მაშინ მთავრდება, როდესაც დარჩენილი მიმდევრობა ზრდადია.
- ვინც ბოლო სვლას გააკეთებს, ის იგებს.

ორივე მოთამაშე ოპტიმალურად თამაშობს, ამიტომ უნდა დავადგინოთ, ვის აქვს მოგების სტრატეგია.

---

## მთავარი იდეა

რადგან `n ≤ 15`, მიმდევრობის თითოეული ელემენტის არსებობა ან არარსებობა შეგვიძლია ბიტმასკით აღვწეროთ.

მაგალითად, თუ გვაქვს:

```text
1 4 3 2
```

ბიტმასკი

```text
1111
```

ნიშნავს, რომ ოთხივე ელემენტი ჯერ კიდევ დარჩენილია.

ხოლო

```text
1011
```

ნიშნავს, რომ მეორე ელემენტი წაშლილია.

ამგვარად თითოეული თამაშის მდგომარეობა შეგვიძლია ერთი მთელი რიცხვით აღვწეროთ.

---

## მდგომარეობის წარმოდგენა

გამოვიყენოთ ბიტმასკი.

ვთქვათ გვაქვს:

```text
5 1 4
```

მაშინ

```text
111
```

ნიშნავს:

```text
5 1 4
```

ხოლო

```text
101
```

ნიშნავს:

```text
5 4
```

რადგან შუა ელემენტი წაშლილია.

---

## ფუნქცია `check()`

ეს ფუნქცია ამოწმებს, დასრულდა თუ არა თამაში.

```cpp
bool check(int mask, vector<int> v)
```

პირველ ეტაპზე ვაგებთ იმ ელემენტებისგან ახალ მიმდევრობას, რომლებიც მოცემულ ბიტმასკში ჯერ კიდევ დარჩენილია.

```cpp
vector<int> seq;

for(int i=0; i<v.size(); i++){
    if(mask & (1 << i)){
        seq.pb(v[i]);
    }
}
```

მაგალითად:

```text
v = [4, 1, 3]

mask = 101
```

მივიღებთ:

```text
seq = [4, 3]
```

---

შემდეგ ვამოწმებთ ზრდადობას.

```cpp
for(int i=0; i<seq.size()-1; i++){
    if(seq[i] > seq[i+1])
        return false;
}
```

თუ სადმე მარცხენა ელემენტი მარჯვენაზე დიდია, მიმდევრობა ჯერ არ არის ზრდადი.

თუ ასეთი წყვილი ვერ ვიპოვეთ:

```cpp
return true;
```

ანუ თამაში დასრულებულია.

---

## დინამიკური პროგრამირება

გამოვიყენოთ:

```cpp
dp[mask]
```

სადაც:

```text
dp[mask] = 1
```

ნიშნავს, რომ ამ მდგომარეობიდან მიმდინარე მოთამაშეს მოგება შეუძლია.

ხოლო

```text
dp[mask] = 0
```

ნიშნავს, რომ ოპტიმალური თამაშის შემთხვევაში მიმდინარე მოთამაშე აგებს.

---

## რეკურსიული ფუნქცია `sol()`

```cpp
bool sol(int mask, vector<int> v)
```

აბრუნებს ინფორმაციას, მოგებულია თუ არა მოცემული მდგომარეობა.

---

### უკვე გამოთვლილი მდგომარეობა

```cpp
if(dp[mask] != -1)
    return dp[mask];
```

თუ ეს მდგომარეობა უკვე გვინახავს, პასუხს პირდაპირ ვაბრუნებთ.

ეს თავიდან გვაცილებს ერთი და იმავე მდგომარეობის მრავალჯერ გამოთვლას.

---

### ბაზური შემთხვევა

```cpp
if(check(mask, v))
    return dp[mask] = 0;
```

თუ მიმდინარე მიმდევრობა უკვე ზრდადია, თამაში დასრულებულია.

ეს ნიშნავს, რომ სვლის გაკეთება აღარ შეიძლება.

შესაბამისად მიმდინარე მოთამაშე აგებს.

---

მაგალითად:

```text
1 2 3
```

თუ შენი სვლაა და ასეთი მდგომარეობა მიიღე, უკვე დაგვიანებულია.

წინა მოთამაშემ ბოლო სვლა გააკეთა და მოიგო.

---

## ყველა შესაძლო სვლის ცდა

ახლა ვცდით თითოეული დარჩენილი ელემენტის წაშლას.

```cpp
for(int i=0; i<v.size(); i++){
```

თუ ელემენტი არსებობს:

```cpp
if(mask & (1 << i))
```

ვშლით მას.

---

### ახალი მდგომარეობის შექმნა

```cpp
int next_mask = mask & (~(1 << i));
```

ეს კოდი ითიშავს `i`-ე ბიტს.

მაგალითად:

```text
mask = 1111
```

თუ ვშლით მესამე ელემენტს:

```text
next_mask = 1011
```

---

## მოგებული სვლის ძებნა

```cpp
if(!sol(next_mask, v))
    return dp[mask] = 1;
```

თუ არსებობს თუნდაც ერთი სვლა, რომლის შემდეგაც მოწინააღმდეგე წაგებულ მდგომარეობაში აღმოჩნდება, მაშინ მიმდინარე მოთამაშე იგებს.

ეს არის თამაშების თეორიის კლასიკური პრინციპი:

```text
არსებობს წაგებულ მდგომარეობაზე გადასვლა
→ მდგომარეობა მოგებულია
```

---

## წაგებული მდგომარეობა

თუ ყველა შესაძლო სვლა შევამოწმეთ და ვერცერთმა ვერ გადაიყვანა მოწინააღმდეგე წაგებულ მდგომარეობაში:

```cpp
return dp[mask] = 0;
```

მაშინ ეს მდგომარეობა წაგებულია.

---

## საწყისი მდგომარეობა

თავდაპირველად ყველა ელემენტი არსებობს.

```cpp
int mask = (1 << n) - 1;
```

მაგალითად:

```text
n = 4
```

მივიღებთ:

```text
1111
```

---

შემდეგ ვამოწმებთ:

```cpp
sol(mask, a)
```

თუ პასუხია:

```cpp
1
```

იგებს ტომი.

```cpp
cout << "Tom\n";
```

წინააღმდეგ შემთხვევაში:

```cpp
cout << "Jerry\n";
```

---

## პატარა მაგალითი

მოცემულია:

```text
3
2 1 3
```

საწყისი მდგომარეობა:

```text
2 1 3
```

ზრდადი არ არის.

ტომს შეუძლია წაშალოს `2`.

მივიღებთ:

```text
1 3
```

რომელიც უკვე ზრდადია.

თამაში დასრულდა და ბოლო სვლა ტომმა გააკეთა.

პასუხი:

```text
Tom
```

---

## რატომ მუშაობს ეს მიდგომა?

რადგან თითოეული მდგომარეობისთვის ზუსტად ვადგენთ:

- არსებობს თუ არა მოგებული სვლა;
- რა მოხდება ოპტიმალური თამაშის შემთხვევაში.

დინამიკური პროგრამირება უზრუნველყოფს, რომ ერთი და იგივე მდგომარეობა მხოლოდ ერთხელ გამოითვალოს.

---

## სირთულე

ბიტმასკების რაოდენობაა:

```text
2^n
```

ხოლო

```text
n ≤ 15
```

ამიტომ მაქსიმუმ:

```text
2^15 = 32768
```

მდგომარეობა არსებობს.

---

### დროითი სირთულე

```text
O(2^n · n²)
```

ყოველი მდგომარეობისთვის შეიძლება:

- ყველა ელემენტის გადამოწმება;
- ზრდადობის შემოწმება.

---

### მეხსიერების სირთულე

```text
O(2^n)
```

რადგან ვინახავთ მხოლოდ `dp` მასივს.

---

## შეჯამება

ამოხსნა ეფუძნება:

1. ბიტმასკით მდგომარეობის აღწერას;
2. რეკურსიას;
3. დინამიკურ პროგრამირებას;
4. თამაშების თეორიის პრინციპს:

```text
თუ არსებობს სვლა, რომელიც მოწინააღმდეგეს წაგებულ მდგომარეობაში ტოვებს,
მიმდინარე მდგომარეობა მოგებულია.
```

ასე შეგვიძლია განვსაზღვროთ, ოპტიმალური თამაშის შემთხვევაში მოიგებს ტომი თუ ჯერი.
