# განმარტება — 688. მაქსიმალური პალინდრომი

## მიდგომა

ეს ტიპიური DP ამოცანაა, სადაც პატარა სტრიქონებიდან ნელ-ნელა უფრო დიდებისკენ მივდივარ. 2D მასივს ვიყენებ, სადაც dp[l][r] მენიშნება 1-ით, თუ [l, r] შუალედი პალინდრომია. იმისთვის, რომ სტრიქონი პალინდრომი იყოს, მისი ბოლოები უნდა ემთხვეოდეს ერთმანეთს და შიგნით რაცაა, ისიც პალინდრომი უნდა იყოს. სიგრძეების მიხედვით ვტრიალებ და ყოველ ჯერზე მაქსიმალურ სიგრძეს ვიმახსოვრებ.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n²) |

## მთავარი დაკვირვებები

- ბაზისი მარტივია: ნებისმიერი ცალკეული სიმბოლო ავტომატურად პალინდრომია, ანუ სიგრძე 1-ია.
- თუ სუბსტრინგის ბოლოები ტოლია და მათ შორის მოქცეული ნაწილი უკვე პალინდრომად გვაქვს მონიშნული, მაშინ მთლიანი შუალედიც პალინდრომია.

## ხშირი შეცდომები

- პატარა სტრიქონების (len=2) ცალკე დამუშავება ან დავიწყება.
- 10^4*10^4 მასივის `main`-ში გამოცხადება, რაც stack overflow-ს გამოიწვევს; გლობალურად უნდა გაიტანო, როგორც მე მაქვს.
