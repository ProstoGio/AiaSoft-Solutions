# განმარტება — 202. უცნაური კიბე

## მიდგომა

ეს ტიპური DP ამოცანაა, სადაც ყოველ საფეხურზე არჩევანს აკეთებ: ან წინა საფეხურიდან მოდიხარ, ან ერთი საფეხურის გამოტოვებით მის წინას უერთდები. რადგან ნებისმიერ დროს შემიძლია ერთი საფეხურის გამოტოვება, მიმდინარე საფეხურის ღირებულებას ვუმატებ უკვე დათვლილი წინა ორი საფეხურიდან მინიმალურს. პირველი ორი საფეხური ცალკე უნდა გაიტანო, რადგან მათთვის 'ორი ნაბიჯით უკან' აღარ არსებობს.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- DP ფორმულა არის მარტივი: d[i] = min(d[i-1], d[i-2]) + a[i]. ფაქტობრივად, ფიბონაჩის პრინციპი გაქვს, ოღონდ ჯამის მაგივრად მინიმუმს იღებ.
- არ გჭირდება მეტი ინფორმაცია, ვიდრე წინა ორი ნაბიჯის მინიმალური ფასი, რადგან ოპტიმალური გზა ლოკალურად წყდება.

## ხშირი შეცდომები

- ინდექსების არევა და 'out of bounds' შეცდომა, თუ ციკლს 0-დან დაიწყებ და მეორე ელემენტს დაუწყებ ძებნას.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ გვაქვს [10, 20, 30, 40]. d[0]=10, d[1]=20. d[2]-ისთვის ვადარებთ 10-ს და 20-ს, ვიღებთ 10-ს და ვუმატებთ 30-ს, გამოდის 40. ბოლოს d[3]-სთვის ვადარებთ 20-ს და 40-ს, ვიღებთ 20-ს და ვუმატებთ 40-ს, საბოლოოდ 60.
