# განმარტება — 248. მასივის შემობრუნება

## მიდგომა

ამ ამოცანაში ველოსიპედის გამოგონება არაა საჭირო. ყველაზე მარტივი გზა არის, რომ თავიდანვე მასივში შეინახო ყველა რიცხვი, მერე კი უბრალოდ ბოლო ინდექსიდან ნულისკენ წამოხვიდე ფორ-ით. პირდაპირ ბეჭდავ ელემენტებს და ეგაა, არანაირი დამატებითი მეხსიერება არ გჭირდება და არც swap-ების წვალება.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- არ ღირს მასივის რეალურად 'შებრუნება', უბრალოდ ინდექსების კლებაში გაყოლა გვაძლევს იგივე შედეგს.
- თუ ინდექსებს სწორად მართავ, O(n) სირთულეში მარტივად აკეთებ საქმეს.

## ხშირი შეცდომები

- ხშირად ავიწყდებათ n-ის შემცირება loop-ის დაწყებამდე, რის გამოც პირველივე წვდომა მასივის საზღვრებს ცდება.
