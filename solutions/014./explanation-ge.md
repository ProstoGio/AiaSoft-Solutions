# განმარტება — 014. ლუწი ხე

## მიდგომა

მთელი მუღამი იმაშია, რომ როგორც კი დავინახავთ ქვეხეს, რომელშიც ლუწი რაოდენობის წვეროა, ეგრევე მივხვდებით — მაგ წიბოს ამოშლა შეიძლება. თუ ქვევით ლუწი დარჩა, ზევითაც ხომ ლუწი დარჩება (მთლიანი n ლუწია), ამიტომ ორივე მხარე 'დაბალანსებული' იქნება. უბრალოდ DFS-ით ვითვლით თითოეული წვეროს ქვემოთ რამდენი ნოდია და თუ ლუწია, პასუხს ვუმატებთ ერთს.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- თუ ქვეხის ზომა ლუწია, მაგ წიბოს მოჭრა ავტომატურად ქმნის ორ ლუწ კომპონენტს.
- Root-ს არ ვთვლით, რადგან მაგის ზემოთ წიბო არ გვაქვს, რომ ამოვჭრათ.

## ხშირი შეცდომები

- Root-ის გათვლა, როცა ეგ წიბო სინამდვილეში არ არსებობს.

## ნაბიჯ-ნაბიჯ ახსნა

წარმოიდგინე ხე: 1-2, 2-3, 2-4. DFS-ით როცა ავდივართ, 3-ს აქვს 1 ნოდი, 4-ს აქვს 1. 2-ს ექნება 3 (თავისი თავი + 3 და 4). 1-ს ექნება 4. ვამოწმებთ 2, 3, 4 წვეროებს: მარტო არცერთი არაა ლუწი, ამიტომ პასუხი 0 იქნება. თუ სტრუქტურა სხვანაირია, უბრალოდ ვითვლით რაოდენობას.
