# განმარტება — 719. მძლავრად ბმული კომპონენტების რაოდენობა

## მიდგომა

მძლავრად ბმული კომპონენტების საპოვნელად Kosaraju-ს ალგორითმი გამოვიყენე, იმიტომ რომ კონცეპტუალურად ძალიან მარტივია. ჯერ ვუშვებთ DFS-ს, რომ წვეროები მათი 'დასრულების დროის' მიხედვით დავალაგოთ, მერე კი გრაფს ვატრიალებთ—ანუ ყველა წიბოს მიმართულებას ვუცვლით. მეორე DFS-ს სწორედ ამ შებრუნებულ გრაფზე ვუშვებთ პირველი ნაბიჯის შედეგად მიღებული თანმიმდევრობით და ზუსტად ეს გვაძლევს საშუალებას, კომპონენტები ერთმანეთისგან დამოუკიდებლად დავთვალოთ.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n²) |

## მთავარი დაკვირვებები

- ადჯასენს მატრიცა აქ O(n²) მეხსიერებას იკავებს, რაც 10,000 წვეროზე საკმაოდ ბევრია, მაგრამ ლიმიტში ჩავდივართ.
- მთავარი 'ტრიუკი' ისაა, რომ პირველი DFS-ის დასრულების დროები გვეუბნება, საიდან დავიწყოთ მეორე DFS, რათა კომპონენტები ერთმანეთში არ აგვერიოს.

## ხშირი შეცდომები

- used მასივების განულება თუ დაგავიწყდა, მთელი ლოგიკა იშლება.
- ადჯასენს მატრიცის ნაცვლად ვექტორების მასივის გამოყენება უფრო ოპტიმალური იქნებოდა, მაგრამ ამ კონკრეტულ ამოცანაში მატრიცაც საკმარისია.

## ნაბიჯ-ნაბიჯ ახსნა

წარმოიდგინე ორი წვერო: A -> B. პირველი DFS წავა A-დან B-ზე, B დაასრულებს, მერე A. t_out მასივში გვექნება [B, A]. გრაფს ვატრიალებთ (B -> A). მეორე DFS-ს ვიწყებთ სიის ბოლოდან, ანუ A-დან, ის აკეთებს ვიზიტს B-ზე და ვხედავთ რომ ეს ერთი კომპონენტია.
