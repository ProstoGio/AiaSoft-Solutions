# განმარტება — 180. სიტყვები

## მიდგომა

პრინციპი მარტივია: ყოველი მომდევნო სიტყვა წინა სიტყვის ბოლო ნაწილს უნდა 'მიეწებოს'. თავიდან ავიღე პირველი სიტყვა, მერე კი თითოეულ ახალ სიტყვაზე ვამოწმებ, ხომ არ არის ის უკვე წინა სიტყვის ნაწილი — თუ არის, მაშინ ეგ 'გამონაკლისი' შემთხვევაა და უბრალოდ ვუმატებ. თუ არ არის, მაშინ ვეძებ ყველაზე გრძელ გადაკვეთას, ანუ რა სიგრძის სუფიქსია 'answer'-ის ბოლოში, რომელიც ახალი სიტყვის პრეფიქსს ემთხვევა. რაც გადაიკვეთა, იმის გამოკლებით ვამატებ დარჩენილ ნაწილს.

## სირთულე

| | |
|-|--|
| დრო | O(N * L^2) |
| მეხსიერება | O(N * L) |

## მთავარი დაკვირვებები

- გამონაკლისების დამუშავება 'find'-ით არის ყველაზე სწრაფი გზა, რადგან მაშინვე ხვდები, საერთოდ საჭიროა თუ არა დამატებითი ლოგიკა.
- გადაკვეთის ძებნისას `min(str1.size(), str2.size())`-მდე ციკლი საკმარისია, რადგან გადაკვეთა სიტყვაზე გრძელი ვერ იქნება.

## ხშირი შეცდომები

- ხშირად ეშლებათ `substr`-ში ინდექსები და არასწორად ჭრიან სიტყვას, რის გამოც ბოლო ასოები ან ორჯერ ჩაიწერება, ან საერთოდ გაქრება.

## ნაბიჯ-ნაბიჯ ახსნა

მაგალითად 'მანქანა' და 'ანაბარი'. 'ანაბარი' არ არის 'მანქანა'-ს ნაწილი. ვეძებთ 'მანქანა'-ს სუფიქსს, რომელიც 'ანაბარი'-ს პრეფიქსია. ეს არის 'ანა'. 'ანაბარი'-დან 'ანა'-ს მოვაჭრით და დაგვრჩება 'ბარი'. საბოლოო ჯამში მივიღებთ 'მანქანაბარი'-ს.
