# განმარტება — 671. ბილეთების გაყიდვა

## მიდგომა

ეს ტიპური DP ამოცანაა, სადაც ყოველ ნაბიჯზე ვინახავთ მინიმალურ დროს, რაც საჭიროა `i`-ური ადამიანის ჩათვლით რიგის მოსაწყობად. თითოეულ პოზიციაზე სამი ვარიანტი გვაქვს: ან თვითონ ყიდულობს ბილეთს, ან წინა ადამიანთან ერთად წყვილდება, ან წინა ორთან ერთად ერთ ჯგუფს ქმნიან. ფორმულა უბრალოდ ირჩევს ამ სამი ვარიანტიდან ყველაზე იაფს და ამატებს მას წინა ნაბიჯის მინიმალურ დროს.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- მთავარია მივხვდეთ, რომ `d[i]`-ს გამოსათვლელად მხოლოდ ბოლო სამ ნაბიჯს ვუყურებთ.
- პირველი სამი ელემენტის დამუშავება ცალკეა საჭირო, თორემ ინდექსებს აცდება და runtime error-ს აგიგდებს.

## ხშირი შეცდომები

- base case-ების არასწორი გათვლა 0, 1 და 2 ინდექსებისთვის.

## ნაბიჯ-ნაბიჯ ახსნა

თუ 3 კაცია, `d[2]`-ში ვამოწმებთ: `d[1]`-ს პლიუს მესამეს ინდივიდუალური დრო, `d[0]`-ს პლიუს მეორე-მესამეს წყვილი, ან პირდაპირ `a[0][2]`, თუ სამივემ ერთად იყიდა.
