# განმარტება — 916. კომპილატორის ახალი ვერსია

## მიდგომა

აქ მთავარია ხაზგასმით გავითვალისწინოთ კომენტარები. ვკითხულობთ ხაზს `getline`-ით და ვამოწმებთ თითოეულ სიმბოლოს. როგორც კი '//'-ს წავაწყდებით, ციკლს ვწყვეტთ, რადგან მაგის მერე რაც იქნება, აღარ გვაინტერესებს. როცა '->'-ს ვიპოვით, უბრალოდ მინუსს წერტილით ვანაცვლებთ და ისარს ვშლით.

## სირთულე

| | |
|-|--|
| დრო | O(N * M) |
| მეხსიერება | O(M) |

## მთავარი დაკვირვებები

- მნიშვნელოვანია, რომ კომენტარის პოვნისთანავე ვწყვეტთ მუშაობას; ასე ავტომატურად ვიცავთ კომენტარის შიგთავსს ცვლილებებისგან.
- erase-ის გამოყენება აქ გამართლებულია, რადგან მაქსიმუმ 60 სიმბოლოა ხაზში და პერფორმანსის პრობლემა არ გვექნება.

## ხშირი შეცდომები

- s[i+1]-ზე წვდომისას index out of bounds-ის აკიდება, თუ ხაზის ბოლო სიმბოლოზე ვართ მისულები.
