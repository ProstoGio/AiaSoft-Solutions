# განმარტება — 022. ღრმად ძებნაში შესვლის და გამოსვლის დრო

## მიდგომა

ამ ამოცანის მთავარი მუღამი იმაშია, რომ თვალყური ადევნო დროს გლობალური ცვლადების გამოყენებით. როგორც კი წვეროში შევდივარ, ვწერ დროს და მერე გადავდივარ მეზობლებზე. გამოსვლის დრო კი მაშინ ფიქსირდება, როცა რეკურსია უკან ბრუნდება, ანუ ყველა მეზობელი უკვე მონახულებული გვაქვს. რადგან პირობა ითხოვს მეზობლებში წვეროს ნომრების ზრდის მიხედვით გადასვლას, თავიდანვე სორტირებაა საჭირო, თორემ რეზულტატი არასწორი იქნება.

## სირთულე

| | |
|-|--|
| დრო | O(m log m) |
| მეხსიერება | O(n + m) |

## მთავარი დაკვირვებები

- t_in იწერება რეკურსიის დასაწყისში, ხოლო t_out იწერება მაშინ, როცა ფუნქცია ასრულებს თავის მუშაობას და ბრუნდება უკან.
- ყოველი წვეროს მეზობლების სორტირება `std::sort`-ით აუცილებელია, რომ მოთხოვნილი თანმიმდევრობა დავიცვათ.

## ხშირი შეცდომები

- თუ გრაფი სრულად არაა ბმული, შეიძლება რაღაც წვეროები გაგვრჩეს, ამიტომ main-ში loop-ის გავლა უსაფრთხოა.

## ნაბიჯ-ნაბიჯ ახსნა

წარმოიდგინე გზა 1-2-3. DFS იწყება 1-ით (t_in=1). გადადის 2-ზე (t_in=2), მერე 3-ზე (t_in=3). 3-ში მეზობლები აღარ არიან, იწერება t_out[3]=1. ბრუნდება 2-ში, იქაც ყველაფერი დამთავრებულია, t_out[2]=2. ბოლოს ბრუნდება 1-ში და t_out[1]=3. ასე მარტივად ივსება დროები.
