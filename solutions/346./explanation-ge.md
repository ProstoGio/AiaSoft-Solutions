# განმარტება — 346. შეტყობინება

## მიდგომა

ეს არის კლასიკური 'decoding' ტიპის ამოცანა. რადგან ქართული ასოები 1-დან 33-მდეა და 0 ჰარებს აღნიშნავს, შეგვიძლია დინამიური დაპროგრამება გამოვიყენოთ. თითოეულ პოზიციაზე ვინახავთ რამდენი გზა არსებობს შეტყობინების ასაწყობად. თუ მიმდინარე ციფრი და მის წინ მდგომი ციფრი ერთად ქმნიან 10-დან 33-მდე რიცხვს, ესე იგი გვაქვს დამატებითი კომბინაციის შანსი და ვაჯამებთ წინა ორ მდგომარეობას.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- 0-ს ვერ შევუერთებთ ვერაფერს, რადგან ასოები 1-დან იწყება, ამიტომ 0-ის დანახვისას მხოლოდ წინა მდგომარეობას ვაგრძელებთ.
- DP-ს ინდექსები ფრთხილად უნდა ავკრიფოთ, რომ მასივის საზღვრები არ დავარღვიოთ დასაწყისში.

## ხშირი შეცდომები

- 0-ის არასწორი დამუშავება, როცა ის კოდის დასაწყისში ან შუაში გვხვდება.
- 33-ზე მეტი რიცხვების შეცდომით დათვლა (მაგალითად, 34-ის ჩათვლა ერთ ასოდ).

## ნაბიჯ-ნაბიჯ ახსნა

მაგალითად, '19': dp[0]=1. '19' შედის 10-33 დიაპაზონში, ამიტომ dp[1] ხდება dp[0]+1=2. ანუ გვაქვს 'აი' (1, 9) და 'ტ' (19).
