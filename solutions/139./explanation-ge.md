# განმარტება — 139. პირველი ნომერი ცელქობა

## მიდგომა

სინამდვილეში, აქ რთული არაფერია - რადგან Swap ოპერაცია ორმხრივია, თუ ერთხელ უკვე გაცვალეს ადგილები, ზუსტად იგივე ნაბიჯების გამეორება მასივს საწყის მდგომარეობას დაუბრუნებს. უბრალოდ მივყვები ციკლით, ვამოწმებ ინდექსი ლუწია თუ არა და ვამოწმებ, გვაქვს თუ არა მომდევნო ელემენტი, რომ 'გადახტომა' არ გამოგვივიდეს. თუ პირობა სრულდება, ვაკეთებ swap-ს და მორჩა.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- Swap ოპერაცია თავისთავად ინვერსიულია, ასე რომ რაც არეულია, იგივე ლოგიკით დალაგდება.
- მთავარია i+1 < n შემოწმება, თორემ ბოლო ელემენტთან მარტივად აირევა ინდექსები.

## ხშირი შეცდომები

- i+1 პირობის გამოტოვება, რამაც შეიძლება მემორისა დააზიანოს თუ მასივის ზომა ლუწია.
