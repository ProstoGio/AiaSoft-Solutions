# განმარტება — 312. უბრალოდ უმოკლესი მანძილი

## მიდგომა

გრაფი დინამიკურია, ამიტომ ყოველ ახალ წიბოზე თავიდან მთელი გრაფის 'დათვლა' საჭირო არ არის. რადგან N მხოლოდ 1000-ია, შეგვიძლია BFS გავუშვათ მხოლოდ მაშინ, როცა ახალი წიბოს დამატება რეალურად ამცირებს მანძილს რომელიმე წვერომდე. `used` მასივით ვინახავთ, რომელი წვეროებია უკვე მიღწევადი და თუ `d[v] + 1 < d[u]` პირობა სრულდება, მაშინ ჩავრთავთ BFS-ს, რომ განახლებული მანძილები მთელ გრაფში გავრცელდეს.

## სირთულე

| | |
|-|--|
| დრო | O(M * (N + M)) |
| მეხსიერება | O(N + M) |

## მთავარი დაკვირვებები

- მთელი გრაფის თავიდან დათვლა არ გვჭირდება, მხოლოდ იმ ცვლილებების გავრცელებაა საკმარისი, რაც მანძილს აუმჯობესებს.
- BFS-ის გაშვება იმ წვეროდან, სადაც ახალი წიბო მთავრდება, საკმარისია იმისთვის, რომ 'ტალღამ' მთელი გრაფი გაიაროს.

## ხშირი შეცდომები

- იმის დავიწყება, რომ ახალი წიბო მხოლოდ მაშინ არის სასარგებლო, თუ მისი საწყისი წვერო უკვე მიღწევადია პირველი წვეროდან.

## ნაბიჯ-ნაბიჯ ახსნა

წარმოიდგინე, ემატება 2->3 წიბო. თუ 2-მდე მისვლა უკვე ვიცით, 3-ის მანძილიც განახლდება და BFS-ს ვაძლევთ იმის საშუალებას, რომ 3-დან გამავალი სხვა გზებიც (მაგალითად 3->4) უფრო მოკლე გახდეს.
