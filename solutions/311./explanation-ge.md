# განმარტება — 311. ძლიერი სამეფო

## მიდგომა

ამოცანა გვთხოვს დავრწმუნდეთ, რომ გრაფი არა მხოლოდ ბმულია, არამედ ყველა წყვილს შორის მანძილი T-ს არ აღემატება. რადგან N მაქსიმუმ 500-ია, Dijkstra-ს N-ჯერ გაშვება მშვენივრად ასწრებს ლიმიტში. თითოეული გაშვებისას ვამოწმებთ, ყველა სხვა ქალაქი თუა მისაწვდომი T დროში; თუ სადმე INF დაგვხვდა ან T-ზე მეტი მანძილი, მაშინვე ვბეჭდავთ 'bno'-ს.

## სირთულე

| | |
|-|--|
| დრო | O(N * (N^2 + M)) |
| მეხსიერება | O(N + M) |

## მთავარი დაკვირვებები

- N=500-ისთვის O(N³) მიახლოებითი სირთულე სრულიად მისაღებია, მით უმეტეს 5 წამიანი ლიმიტის გათვალისწინებით.
- Dijkstra-ს სტანდარტული იმპლემენტაცია Priority Queue-ს გარეშეც მშვენივრად მუშაობს ამ ზომის გრაფებზე.

## ხშირი შეცდომები

- INF-ის არასწორად შერჩევა; რადგან T თავად 10^9-ია, INF აუცილებლად უფრო დიდი უნდა იყოს (მაგალითად, 1e15 ან უბრალოდ საკმარისად დიდი რიცხვი).
- გრაფის არასრულად შემოწმება - მხოლოდ ზოგიერთი წყვილის შემოწმება არ გვაძლევს გარანტიას.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ, გვაქვს 3 ქალაქი და T=10. გავუშვებთ Dijkstra(1)-ს, თუ d[2]=5 და d[3]=12, მაშინვე ვხვდებით, რომ მე-3 ქალაქამდე მისვლა დროში ვერ ეტევა და პასუხი 'bno'-ა.
