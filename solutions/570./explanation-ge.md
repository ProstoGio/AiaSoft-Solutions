# განმარტება — 570. უახლოესი რიცხვი

## მიდგომა

რადგან მასივი დალაგებულია, წრფივი ძებნა 10^5 ელემენტიან მასივში ძალიან ნელი იქნებოდა, ამიტომ Binary Search გამოვიყენე. იდეა მარტივია: ვავიწროებ ინტერვალს მანამ, სანამ ხელში მხოლოდ ორი მეზობელი რიცხვი არ დამრჩება, რომელთა შორისაც უნდა იყოს საძიებელი x. ბოლოს უბრალოდ ვამოწმებ, რომელია უფრო ახლოს x-თან და იმას ვაბეჭდინებ.

## სირთულე

| | |
|-|--|
| დრო | O(Q log N) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- while(r-l > 1) პირობა იდეალურია, იმიტომ რომ ძებნას წყვეტს მაშინვე, როგორც კი ორ მეზობელ ელემენტამდე დავალ.
- პირობაში მითითებულია, რომ თანაბარი დაშორებისას მინიმალური უნდა ავირჩიოთ, ამიტომ l-ს (მარცხენა ინდექსს) ვანიჭებ უპირატესობას.

## ხშირი შეცდომები

- ზოგჯერ ავიწყდებათ საზღვრების დამუშავება, როცა x მთელ მასივზე პატარაა ან დიდია, თუმცა ამ ლოგიკით l და r ისედაც მასივის საზღვრებში რჩებიან.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ მასივია [10, 20, 30] და ვეძებთ 22-ს. Binary search-ით მივალთ l=1 (20) და r=2 (30)-მდე. 22-20=2, ხოლო 30-22=8; 2 < 8, ამიტომ პასუხია 20.
