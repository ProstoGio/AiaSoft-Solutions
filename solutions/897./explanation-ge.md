# განმარტება — 897. სად არის პირველად?

## მიდგომა

აქ მთავარი პრობლემა ისაა, რომ Q და N მილიონამდე ადის, ანუ ყოველი შეკითხვისთვის მასივში ძებნა რომ დავიწყო, TLE გარანტირებულია. იდეა მარტივია: წინასწარ შევქმნათ 1,000,001 ზომის მასივი, სადაც შევინახავთ თითოეული რიცხვის პირველ პოზიციას. როგორც კი პირველად შევხვდებით რიცხვს, მაშინვე ჩავწეროთ მისი ინდექსი, ხოლო მეორეჯერ ნახვისას უბრალოდ იგნორი გავუკეთოთ, რადგან ხომ უკვე ვიცით პირველი პოზიცია.

## სირთულე

| | |
|-|--|
| დრო | O(N + Q) |
| მეხსიერება | O(MAX_VAL) |

## მთავარი დაკვირვებები

- გამოიყენე სტატიკური მასივი ან ვექტორი ინდექსების შესანახად, ეს ო(1)-ში გაძლევს პასუხს.
- ინდექსები თავიდანვე -1-ით გაავსე, რომ თუ რიცხვი არ გვხვდება, პირდაპირ გამოიტანო -1.

## ხშირი შეცდომები

- ყოველი შეკითხვისთვის `find`-ის გამოყენება ან `for` ციკლით ძებნა.
