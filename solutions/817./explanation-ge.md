# განმარტება — 817. მეგობრული წევრები

## მიდგომა

აქ მთავარია მივხვდეთ, რომ მეგობრული სამეულები გადაფარვადი უნდა იყოს. თუ გვაქვს '15, 15, 15, 15', პირველი სამეული არის ინდექსებზე [0,1,2], ხოლო მეორე - [1,2,3]. ანუ, არანაირი დამატებითი ლოგიკა არ გვჭირდება, უბრალოდ ვამოწმებთ ყოველი `i`-სთვის, ტოლია თუ არა `i`, `i+1` და `i+2`. თუ ტოლია, counter-ს ვზრდით და ეგაა.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- რადგან ამოცანაში სამეულები გადაფარვადი უნდა იყოს, მარტივი for loop იდეალურად მუშაობს.
- ციკლი უნდა გავაჩეროთ n-2-ზე, თორემ memory-ს დავარდნა ან garbage მნიშვნელობების შედარება მოგვიწევს.

## ხშირი შეცდომები

- ინდექსების არასწორი კონტროლი, რამაც შეიძლება 'segmentation fault' გამოიწვიოს.
- სამეულის პოვნის მერე i-ს ხელით გაზრდა, რაც გადამფარავ შემთხვევებს გამოგვატოვებინებს.
