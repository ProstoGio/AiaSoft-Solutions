# განმარტება — 120. ყუთები

## მიდგომა

ეს პრობლემა რეალურად LIS-ის (Longest Increasing Subsequence) ვარიაციაა, ოღონდ ოდნავ შეფუთული. რადგან ყუთების თანმიმდევრობა ფიქსირებულია და ვერ შევცვლით, მთელი აზრი იმაშია, რომ სწორად შევარჩიოთ რომელი ყუთები დავდოთ ერთმანეთზე, რომ მაქსიმალურ სიმაღლეს მივაღწიოთ. DP მასივში ვინახავ მაქსიმალურ სიმაღლეს, რომელიც კონკრეტულ ყუთთან მთავრდება. ყოველი ყუთისთვის გადავუყვები ყველა წინა ყუთს და თუ ახალი ყუთი ძველზე პატარაა ან ტოლი, ანუ ზემოდან ეტევა, მაშინ განვაახლებ ამ პოზიციის სიმაღლეს. ყველაზე მოსახერხებელი მომენტია გვერდების დალაგება - `swap`-ით ვაკეთებ ისე, რომ ყოველთვის `a[i][1] <= a[i][2]` იყოს, ამით შედარებას ძალიან ვამარტივებ.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- ყოველთვის დაალაგე ყუთის ფუძის გვერდები: პატარა გვერდი პირველ ადგილზე, დიდი მეორეზე. ამით შედარებისას აღარ გჭირდებათ ორივე ორიენტაციის განხილვა.
- DP-ს ლოგიკა მარტივია: `dp[i]` არის `i`-ური ყუთის სიმაღლე პლუს `dp[j]`-ების მაქსიმუმი, სადაც `j < i` და `i` ეტევა `j`-ზე.

## ხშირი შეცდომები

- ყუთების დახარისხება ზომების მიხედვით. გვავიწყდება, რომ ამოღების დროს რიგითობის დარღვევა არ შეიძლება, ამიტომ მასივი უნდა დავტოვოთ ისე, როგორც შემოვიდა.
- დაგვავიწყდეს `dp[i]`-ს ინიციალიზება თავად ყუთის სიმაღლით, როცა მის ქვეშ ვერცერთი ყუთი ვერ დგება.

## ნაბიჯ-ნაბიჯ ახსნა

მაგალითად, თუ გვაქვს ყუთი i და ვამოწმებთ ყუთ j-ს (სადაც j < i), ვამოწმებთ პირობას `a[i][1] <= a[j][1]` და `a[i][2] <= a[j][2]`. თუ ეს მართალია, `dp[i]` გახდება `max(dp[i], dp[j] + a[i][0])`. ბოლოს კი უბრალოდ მთელი `dp` მასივიდან ვიღებთ მაქსიმუმს.
