# განმარტება — 782. სოკო

## მიდგომა

ეს ამოცანა ჩვეულებრივი მათემატიკური გაყოფაა. რადგან კალათებს ნახევრებს ვერ ვიყიდით, თუ სოკოების რაოდენობა კალათის ტევადობაზე ნაშთით იყოფა, ერთი დამატებითი კალათა აუცილებლად დაგვჭირდება. ყველაზე მარტივი გზაა შევამოწმოთ m%n ნაშთი: თუ 0-ია, პირდაპირ m/n-ს ვბეჭდავთ, სხვა შემთხვევაში m/n-ს ვუმატებთ ერთს.

## სირთულე

| | |
|-|--|
| დრო | O(1) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- C++-ში მთელი რიცხვების გაყოფა (integer division) ყოველთვის ქვემოთ ამრგვალებს, ამიტომ ნაშთის შემოწმება გვაძლევს 'ჭერის' (ceiling) ფუნქციის ეფექტს.
- პირობითად, თუ სოკო საერთოდ არ გვაქვს, კოდი 0-ს დააბრუნებს, რაც სწორია.

## ხშირი შეცდომები

- ნაშთის იგნორირება და მხოლოდ m/n-ის დაწერა, რითაც ბოლო კალათა დაიკარგება.
