# განმარტება — 301. ლაბირინთი

## მიდგომა

როცა ლაბირინთში უმოკლეს გზას გთხოვენ და თან სვლები უწონოა, BFS-ს ჯობია არაფერია. უბრალოდ ვიწყებ S წერტილიდან, ჩავაგდებ queue-ში და თითოეულ უჯრაზე ვიანგარიშებ სვლების რაოდენობას მანამდე, სანამ F-ს არ მივადგები. რადგან BFS ფენებად მუშაობს, პირველი შემთხვევა, როცა F-ზე დავეშვები, გარანტირებულად იქნება ყველაზე ოპტიმალური პასუხი.

## სირთულე

| | |
|-|--|
| დრო | O(N * M) |
| მეხსიერება | O(N * M) |

## მთავარი დაკვირვებები

- BFS-ის რიგი (queue) ისეა მოწყობილი, რომ მუდამ უახლოეს უჯრებს ამუშავებს, ამიტომ პირველივე მიგნება F-ზე არის ოპტიმალური.
- used მასივი აუცილებელია, რომ არ დავციკლოთ ერთსა და იმავე უჯრებზე.

## ხშირი შეცდომები

- საზღვრების შემოწმების გამორჩენა (მაგალითად, -1 ინდექსით წვდომა), რაც ხშირად იწვევს Runtime Error-ს.
- თუ უკვე ნამყოფი ხარ უჯრაზე, მეორედ არ უნდა გადათვალო დისტანცია.
