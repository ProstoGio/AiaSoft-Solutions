# განმარტება — 542. ჯადოქარი ბიტუნა

## მიდგომა

XOR-ების ჯამი 1-დან N-მდე 4-ციკლიანი პატერნით ითვლება. თუ დააკვირდები, შედეგები ყოველთვის რიგდება ასე: N, 1, N+1, 0. რადგან N 10^35-ია, `long long`-ში ნამდვილად არ ჩაეტევა, ამიტომ სტრიქონად შემომყავს და ბოლო ორი ციფრის მიხედვით ვადგენ 4-ზე ნაშთს.

## სირთულე

| | |
|-|--|
| დრო | O(log N) |
| მეხსიერება | O(log N) |

## მთავარი დაკვირვებები

- XOR-ის თანმიმდევრობა ციკლურია: 4-ზე ნაშთის მიხედვით ვიგებთ, რა უნდა დავაბრუნოთ.
- სტრიქონით მუშაობა მარტივი გზაა დიდი რიცხვების დასამუშავებლად, ყოველგვარი BigInt-ის ბიბლიოთეკების გარეშე.

## ხშირი შეცდომები

- სტანდარტული ტიპების (int, long long) გამოყენება, რაც N-ის სიდიდის გამო მყისიერად ჩააგდებს პროგრამას.

## ნაბიჯ-ნაბიჯ ახსნა

ავიღოთ N=6. ბოლო ორი ციფრია 06, 6-ს გაყოფილი 4-ზე ნაშთი არის 2. წესის მიხედვით, თუ ნაშთი 2-ია, პასუხი უნდა იყოს N+1, ანუ 7.
