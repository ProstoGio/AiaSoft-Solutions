# განმარტება — 253. დიპლომები

## მიდგომა

თავიდან შეიძლება იფიქრო, რომ რაღაც მათემატიკური ფორმულაა საჭირო დიპლომების ჩასალაგებლად, მაგრამ სინამდვილეში ეს Binary Search-ის კლასიკური შემთხვევაა. რაც უფრო დიდია კვადრატის გვერდი, მით მეტი დიპლომი ეტევა მასში, ანუ გვაქვს მონოტონური ფუნქცია. ვიღებთ დიაპაზონს 1-დან მაქსიმალურ შესაძლო ზომამდე (როცა ყველა დიპლომს ერთ გრძელ ხაზად ვალაგებთ) და ვამოწმებთ შუა წერტილს. თუ (m/w) * (m/h) ჯამში იმაზე მეტი ან ტოლია, რაც გვჭირდება, მაშინ ვცდილობთ უფრო პატარა ზომა ვიპოვოთ.

## სირთულე

| | |
|-|--|
| დრო | O(log(max(w, h) * n)) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- კვადრატის გვერდის სიგრძე 'm'-ზე გაყოფით ზუსტად ვადგენთ, რამდენი დიპლომი ეტევა ჰორიზონტალურად და ვერტიკალურად.
- Binary search-ის დროს ზედა ზღვრად max(w, h) * n სრულიად საკმარისია და არ დაგვჭირდება რთული მათემატიკური ფორმულები.

## ხშირი შეცდომები

- overflow-ის პრობლემა `long long`-ის არ გამოყენების გამო, განსაკუთრებით მაშინ, როცა N და w/h დიდი მნიშვნელობებია.

## ნაბიჯ-ნაბიჯ ახსნა

წარმოიდგინე N=10, w=2, h=3. ვეძებთ მინიმალურ m-ს, სადაც (m/2)*(m/3) >= 10. ვამოწმებთ m-ის შუა მნიშვნელობებს. მაგალითად, m=7-ისთვის გვაქვს 3*2=6 დიპლომი, რაც ნაკლებია 10-ზე, ამიტომ l=m+1. ვზრდით m-ს მანამ, სანამ პირობა არ შესრულდება.
