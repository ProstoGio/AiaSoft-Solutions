# განმარტება — 786. მარტივი რიცხვები N-მდე

## მიდგომა

აქ უბრალოდ N-მდე ყველა რიცხვს ვამოწმებ, არის თუ არა მარტივი. მარტივი რიცხვის იდეა ხომ მარტივია: მხოლოდ 1-ზე და საკუთარ თავზე უნდა იყოფოდეს. ამიტომ, თითოეულ რიცხვს ვუყურებ და ვამოწმებ გამყოფებს 1-დან მის კვადრატულ ფესვამდე. თუ გამყოფების რაოდენობა ზუსტად 2 გამოდის, ესე იგი ეგ რიცხვი მარტივია და ეკრანზე გამოგვაქვს.

## სირთულე

| | |
|-|--|
| დრო | O(n * sqrt(n)) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- საკმარისია გამყოფები მხოლოდ ფესვამდე ვეძებოთ, რადგან თუ რიცხვს აქვს გამყოფი ფესვზე მეტი, მაშინ მას მეორე გამყოფი აუცილებლად ექნება ფესვზე ნაკლები.
- კოდში `k` ცვლადი ითვლის გამყოფებს, და რადგან 1 ყოველთვის გამყოფია, მარტივი რიცხვისთვის `k` უნდა იყოს ზუსტად 2.

## ხშირი შეცდომები

- ხშირად ავიწყდებათ, რომ 1 მარტივი რიცხვი არაა, მაგრამ ჩემი ციკლი 2-დან იწყება და პრობლემა იხსნება.
- თუ გამყოფებს N-მდე შეამოწმებ, Time Limit-ში ვერ ჩაეტევი, ამიტომ `p*p <= x` ოპტიმიზაცია აუცილებელია.
