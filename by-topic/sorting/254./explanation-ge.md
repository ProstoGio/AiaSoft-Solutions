# განმარტება — 254. მასივის დალაგება - 1

## მიდგომა

N მაქსიმუმ 100-ია, ამიტომ ზედმეტი ფილოსოფია და 'სწრაფი' სორტირებები არ გვჭირდება. Bubble sort-ის რეალიზება ყველაზე მარტივი და სწრაფია, კოდიც მოკლე გამოდის. უბრალოდ ვამოწმებ მეზობელ ელემენტებს და თუ მარცხენა მარჯვენაზე დიდია, ადგილებს ვუცვლი. ასე ნელ-ნელა, ყველაზე დიდი ელემენტები 'ტივტივდებიან' ბოლოსკენ.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- როცა შეზღუდვები ასეთი მცირეა, Bubble sort-ის დაწერა წამების საქმეა და სრულიად საკმარისია.
- შიდა ციკლის პირობა j < n-i-1 ზუსტად იმას აკეთებს, რომ უკვე დალაგებულ ელემენტებს აღარ ეხება.

## ხშირი შეცდომები

- ზოგჯერ ხალხი ცდილობს std::sort-ის გარეშე რაღაც ზედმეტად რთული დაწეროს, როცა ყველაზე ბანალური ვარიანტიც მუშაობს.
