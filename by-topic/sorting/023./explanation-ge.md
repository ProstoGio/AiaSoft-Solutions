# განმარტება — 023. ინვერსიების რაოდენობა (მარტივი)

## მიდგომა

რადგან n მაქსიმუმ 1000-ია, ბრუტფორსული მიდგომა მშვენივრად მუშაობს. უბრალოდ ვუშვებ ორ ციკლს, რომლებიც ამოწმებენ ყველა შესაძლო წყვილს (i, j). თუ პირობა i < j და a[i] > a[j) სრულდება, ვზრდი ინვერსიების რაოდენობას.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- როცა n 1000-მდეა, O(n²) სირთულე სრულიად საკმარისია და არაფერი უფრო რთულის მოფიქრება არ არის საჭირო.
- ინვერსიის განმარტება პირდაპირ გვეუბნება, რომ თითოეული წყვილი უნდა შევამოწმოთ.

## ხშირი შეცდომები

- i < j პირობის გამოტოვება და არასწორი რაოდენობის მიღება.
- შემთხვევითი ინდექსების არასწორად არჩევა.

## ნაბიჯ-ნაბიჯ ახსნა

მთავარი იდეა ისაა, რომ ვიღებ მასივს და ორმაგი ციკლით გავდივარ ყველა შესაძლო კომბინაციას. გარე ციკლით ვირჩევ i-ს, შიდა ციკლით - j-ს, რომელიც i-ს შემდეგ მოდის. თუ მარცხენა ელემენტი მარჯვენაზე მეტია, ეს ინვერსიაა, ვინახავ ამას counter-ში და ბოლოს ვბეჭდავ.
