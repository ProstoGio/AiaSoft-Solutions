# განმარტება — 425. სიმრავლეები

## მიდგომა

სიმრავლეების დროს ყველაზე თავისმტეხი მომენტი ისაა, რომ {a, b} და {b, a} ერთი და იგივეა. ამიტომ, გადავწყვიტე ყველა სიმრავლე სტანდარტულ ფორმამდე დამეყვანა. რეკურსიულად ვშლი სიმრავლეს მის შიდა ელემენტებად, ვალაგებ მათ და ისე ვაწყობ უკან სტრიქონად. საბოლოო ჯამში, თუ ორი სიმრავლე იდენტურია, მათი 'ნორმალიზებული' სტრიქონებიც ზუსტად ერთნაირი იქნება და map-ში მარტივად დავითვლი სიხშირეებს.

## სირთულე

| | |
|-|--|
| დრო | O(n² log n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- ნორმალიზაცია ყველაზე მაგარი ხრიკია: რეკურსიულად დალაგებული სტრიქონი სიმრავლის 'ანაბეჭდს' ჰგავს.
- სტეკი აქ იდეალურად მუშაობს, რადგან ზუსტად გვეუბნება, როდის სრულდება მიმდინარე სიმრავლის შიდა ელემენტი.

## ხშირი შეცდომები

- თუ ელემენტებს რეკურსიულად არ დაალაგებ, map-ში ვერ გაარჩევ ერთმანეთისგან იდენტურ, მაგრამ განსხვავებული თანმიმდევრობის სიმრავლეებს.

## ნაბიჯ-ნაბიჯ ახსნა

მაგალითად, თუ გაქვს {{}{}}, სტეკით ვპოულობთ ორ ელემენტს: {} და {}. რადგან ორივე {}-ია, მათი დალაგება არაფერს ცვლის. საბოლოოდ მივიღებთ სტრიქონს, სადაც ელემენტები სტანდარტულად არის დალაგებული და map-ში ამ სტრიქონის სიხშირე გაიზრდება.
