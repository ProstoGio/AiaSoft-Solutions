# განმარტება — 666. ლურსმნები

## მიდგომა

ყველაზე მთავარი აქ დალაგებაა - სანამ სორტირებას არ გააკეთებ, ნაბიჯებს ვერ გაწერ. როგორც კი ლურსმნები რიგში ჩადგება, ვხედავთ, რომ თითოეულ ლურსმანს ან წინა ლურსმანთან უნდა ჰქონდეს ძაფი, ან კიდევ უფრო წინასთან. DP-ს ლოგიკა მარტივია: ყოველი ახალი ლურსმნისთვის ვირჩევთ, ჯობია წინა საფეხურის მინიმუმს დავუმატოთ მანძილი, თუ ორი ნაბიჯით უკანასკნელს. საბოლოოდ, d[n]-ში გვექნება ის ოპტიმალური პასუხი, რაც გვინდა.

## სირთულე

| | |
|-|--|
| დრო | O(n log n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- სორტირება აუცილებელია, თორემ DP-ს ლოგიკა აირევა და ვერ გაიგებ რომელი ორი ლურსმანი დააკავშირო.
- ხუთი ლურსმნის ზემოთ უკვე სტანდარტული რეკურენსია მუშაობს, რაც კოდს ძალიან ამარტივებს.

## ხშირი შეცდომები

- თუ დალაგებას დაგავიწყებს, ლოგიკა მთლიანად ინგრევა.
- საწყისი მნიშვნელობების (d[1]-d[4]) არასწორად გაწერა.

## ნაბიჯ-ნაბიჯ ახსნა

მაგალითად, თუ გაქვს 5 ლურსმანი, d[4]-მდე ხელით გათვლილი კავშირები გჭირდება, მერე კი უბრალოდ ციკლი მიყვება მინიმალურს და ავტომატურად ითვლის ყველაზე მოკლე ძაფს.
