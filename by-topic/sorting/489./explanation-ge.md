# განმარტება — 489. დალაგება სიხშირით

## მიდგომა

რადგან რიცხვები -100-დან 100-მდე მერყეობს, გამოვიყენე 201-ელემენტიანი მასივი სიხშირეების დასათვლელად, ოღონდ 100-იანი ოფსეტით, რომ უარყოფითი ინდექსების პრობლემა არ მქონოდა. ეს წყვილები (სიხშირე, მნიშვნელობა) გადავიტანე ვექტორში და დავასორტირე ჩემი პირობის მიხედვით. სორტირებისას მთავარია, სიხშირის მიხედვით ზრდადობა დავიცვათ, ხოლო თანაბარი სიხშირის დროს - მნიშვნელობის მიხედვით კლებადობა. ბოლოს უბრალოდ ვბეჭდავ თითოეულ რიცხვს იმდენჯერ, რამდენჯერაც გვხვდება.

## სირთულე

| | |
|-|--|
| დრო | O(N log N) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- 100-იანი ოფსეტი გვაძლევს საშუალებას მასივი გამოვიყენოთ ჰეშ-ტაბლად.
- სორტირებისას `a.second > b.second` პირობა ზუსტად აკმაყოფილებს უდიდესი მნიშვნელობის პრიორიტეტს.

## ხშირი შეცდომები

- ბეჭდვის დროს 100-ის გამოკლების დავიწყება.
- სიხშირის დალაგებისას ორიგინალი მნიშვნელობების არევა.

## ნაბიჯ-ნაბიჯ ახსნა

თავიდან ვითვლით თითოეული რიცხვის სიხშირეს. შემდეგ ამ მონაცემებს ვინახავთ `pair`-ების ვექტორში. `sort`-ის ფუნქციაში ვწერთ ლამბდას, რომელიც ჯერ სიხშირეს ადარებს, ხოლო თუ სიხშირეები ტოლია, მაშინ მნიშვნელობას ვადარებთ კლებადობით. ბოლო ეტაპზე ვუვლით დალაგებულ ვექტორს და თითოეულ რიცხვს ვბეჭდავთ იმდენჯერ, რამდენიც არის მისი სიხშირე, ოღონდ არ დაგვავიწყდეს 100-ის გამოკლება, რომელიც დასაწყისში მივუმატეთ.
