# განმარტება — 423. ორი ერთიანი

## მიდგომა

რადგან N მაქსიმუმ 10^9-ია, ვიცით რომ პასუხი 2^30-ზე მეტი ვერ იქნება, ამიტომ უბრალოდ ყველა შესაძლო წყვილი 2^i + 2^j უნდა ავაწყოთ. ორი nested loop-ი სრულიად საკმარისია, სადაც i და j ბიტების პოზიციებს აღნიშნავენ. თუ ჯამი N-ს აჭარბებს, ციკლს ვწყვეტთ, რადგან უფრო დიდ რიცხვებს აღარ აქვს აზრი.

## სირთულე

| | |
|-|--|
| დრო | O(log² N) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- ორიანიანიანი რიცხვი ყოველთვის 2^i + 2^j სახით წარმოიდგინება, სადაც i > j.
- 30 ბიტი სრულიად საკმარისია 10^9-მდე ყველა კომბინაციის დასათვლელად.

## ხშირი შეცდომები

- int ტიპის overflow-ის დავიწყება, როცა ბიტებს წევთ (თუმცა 30-მდე ჯერჯერობით არც არის პრობლემა).
