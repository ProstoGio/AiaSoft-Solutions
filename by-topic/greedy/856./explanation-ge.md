# განმარტება — 856. მოლაშქრე კენგურუ

## მიდგომა

ამოცანის არსი იმაშია, რომ კენგურუ ყოველთვის ეძებს შემდეგ უახლოეს მთას, რომელიც წინა ნამყოფ მთაზე მაღალია. ჯერ ვპოულობ მაქსიმალური მთის ინდექსს, რომ ვიცოდე სად ჩერდება პროცესი. შემდეგ უბრალოდ მივუყვები მასივს და ვინახავ ბოლო მთის სიმაღლეს, რომელზეც კენგურუ ახტა. როგორც კი მასზე მაღალს ვნახავ, ნახტომების რაოდენობას ვზრდი და ახალ მთაზე გადავდივარ.

## სირთულე

| | |
|-|--|
| დრო | O(N) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- ხარბი (greedy) მიდგომა: ყოველთვის პირველივე მაღალი მთაა პრიორიტეტული.
- პირველი მთა თავიდანვე ათვლილია, ამიტომ საწყისი პოზიცია 1-ია.

## ხშირი შეცდომები

- ნახტომების ათვლის არევა (მაგალითად, 0-დან დაწყება).
- O(N^2) მიდგომის გამოყენება, რაც არ არის საჭირო.

## ნაბიჯ-ნაბიჯ ახსნა

ჯერ ვკითხულობ მთების სიმაღლეებს და ვინახავ ვექტორში. პარალელურად ვპოულობ უმაღლესი მთის ინდექსს. შემდეგ ვიწყებ ციკლს: `prev` ცვლადით ვინახავ ინდექსს, სადაც ბოლოს ვიყავი. თუ `a[i]` მეტია `a[prev]`-ზე, ვზრდი ნახტომების რაოდენობას და ვანახლებ `prev`-ს. ბოლოს, როცა ციკლი მაქსიმუმამდე მივა, ვბეჭდავ `d[idx]`-ს, სადაც ნახტომების ჯამია დაგროვილი.
