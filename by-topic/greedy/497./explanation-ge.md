# განმარტება — 497. ოთხი ფერის ბურთულა

## მიდგომა

ეს ტიპური Pigeonhole Principle-ის ამოცანაა. იმისთვის, რომ აუცილებლად გვქონდეს ყველა ფერის ბურთულა, უნდა წარმოვიდგინოთ ყველაზე ცუდი სცენარი, სადაც მაქსიმალურად ბევრ ბურთულას ვიღებთ ისე, რომ რომელიმე ერთი ფერი მაინც გვაკლია. ამისთვის ავიღებთ სამ ყველაზე დიდ რაოდენობას და დავუმატებთ ერთს. ჩემი კოდი ამას მარტივად აკეთებს: ჯამს აკლებს ყველაზე პატარა მნიშვნელობას და უმატებს ერთს.

## სირთულე

| | |
|-|--|
| დრო | O(1) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- ყველაზე ცუდი სცენარი არის ის, როცა სამ ფერს სრულად ამოვიღებთ და ბოლო ფერი ჯერ კიდევ არ გვექნება.
- ფორმულაა (A+B+C+D) - min(A,B,C,D) + 1.

## ხშირი შეცდომები

- ბოლოში +1-ის დავიწყება.
- სიმულაციის გაკეთების მცდელობა, მაშინ როცა მათემატიკური ფორმულა გაცილებით მარტივია.

## ნაბიჯ-ნაბიჯ ახსნა

თავიდან ვკითხულობთ ოთხივე ფერის რაოდენობას. თუ ამოვიღებთ ყველა ბურთულას გარდა ყველაზე ცოტა ფერის ბურთულებისა და დავუმატებთ კიდევ ერთს, გარანტირებულად ამოვიღებთ იმ ერთ ბურთულასაც, რომელიც გვაკლდა. ამიტომ, ჯამს ვაკლებთ მინიმალურს და ვუმატებთ ერთს.
