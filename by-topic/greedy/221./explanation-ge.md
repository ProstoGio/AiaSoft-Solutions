# განმარტება — 221. ქალაქი ჩე

## მიდგომა

თავიდან შეიძლება მოგინდეს O(n²) მიდგომა, მაგრამ 300 ათასი ელემენტისთვის ეგ პირდაპირ TLE-ს მოგცემს. ამიტომ ვიყენებთ two-pointer ტექნიკას. მარცხენა პოინტერს ვამოძრავებთ თითოეულ ძეგლზე, ხოლო მარჯვენა პოინტერი მხოლოდ წინ მიდის და გვეუბნება, სად იწყება ისეთი ძეგლები, რომლებიც ჩვენს პირობას აკმაყოფილებენ. საბოლოოდ, ყოველი l-ისთვის, n-rt რაოდენობის წყვილი გვაქვს, რაც მთლიანად O(n)-ში გვახურდავებს საქმეს.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- რადგან მონაცემები უკვე დალაგებულია, დამატებითი სორტირება არ გვჭირდება, რაც დროს გვინახავს.
- როცა v[rt] - v[l] > r, ეს ნიშნავს რომ rt-დან ბოლომდე ყველა ძეგლი 'უსაფრთხოა' გოგონასთვის, ანუ მანძილი r-ზე მეტია.

## ხშირი შეცდომები

- O(n²) მიდგომის არჩევა, რაც ნელი იქნება.
- ინდექსებთან მუშაობისას 1-ით ცდომილება, მაგალითად (n-rt)-ის ნაცვლად (n-rt+1)-ის დაწერა.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ, გვაქვს [1, 3, 5, 8] და r=3. l=0 (v[l]=1), rt ადის 2-მდე (v[rt]=5). 5-1=4 > 3, ანუ 5 და 8 ვარგა. cnt ემატება n-rt = 4-2=2. შემდეგ l=1 (v[l]=3), rt იგივე რჩება, 8-3=5 > 3, ანუ 8 ვარგა. cnt ემატება 4-2=2 (სულ 3 წყვილი).
