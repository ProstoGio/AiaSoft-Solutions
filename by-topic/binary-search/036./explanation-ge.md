# განმარტება — 036. ასტერიქსი და ობელიქსი

## მიდგომა

ეს ამოცანა არის კლასიკური binary search on answer. რადგან რაც უფრო მეტი დრო გადის, მით მეტ ტორტს ჭამენ, ჭამის პროცესი მონოტონურია. ჩვენი მიზანია ვიპოვოთ მინიმალური დრო 'm', სადაც ასტერიქსის (m/a) და ობელიქსის (m/b) ნაჭამი ტორტების ჯამი >= N. თუ ეს პირობა სრულდება, ვცდილობთ დრო შევამციროთ, თუ არა - ვზრდით.

## სირთულე

| | |
|-|--|
| დრო | O(log(N * min(a, b))) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- f(t) = floor(t/a) + floor(t/b) ფუნქცია არის მონოტონურად მზარდი.
- binary search-ის ზედა საზღვარი N * min(a, b) სრულიად საკმარისია.

## ხშირი შეცდომები

- დიდი რიცხვების გამო არასწორი საზღვრების (bounds) არჩევა.
- overflow-ის თავიდან ასაცილებლად long long-ის არგამოყენება.

## ნაბიჯ-ნაბიჯ ახსნა

თავიდან 'l' მაქვს 0, ხოლო 'r' არის 10^18. ციკლის შიგნით ვითვლი, რამდენ ტორტს შეჭამენ 'm' დროში. თუ ჯამი >= N, ესე იგი ეს დრო შესაძლო კანდიდატია, ამიტომ ვინახავ მას და 'r'-ს ვწევ მარცხნივ. წინააღმდეგ შემთხვევაში 'l'-ს ვზრდი, რათა მეტი დრო მივცეთ ჭამისთვის.
