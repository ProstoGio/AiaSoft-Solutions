# განმარტება — 025. მასივი და მისი ორი ელემენტი

## მიდგომა

პირდაპირ რომ ავიღოთ და ყველა შესაძლო წყვილი შევამოწმოთ, O(n²) სირთულე გამოვა, რაც 10⁵-იანი მასივისთვის ნაღდად ვერ გაქაჩავს დროში. ჯობია ჯერ მასივი დავასორტიროთ, მერე კი თითოეული ელემენტისთვის binary search-ით ვეძებოთ მისი 'წყვილი', ანუ s-დან გამოკლებული რიცხვი. ეს მიდგომა O(n log n)-ში გვატევს, რაც სრულიად საკმარისია 1.0 წამისთვის.

## სირთულე

| | |
|-|--|
| დრო | O(n log n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- სორტირება ყველაზე მნიშვნელოვანი ნაწილია, რადგან ის გვაძლევს საშუალებას binary search გამოვიყენოთ.
- s - a[i] ლოგიკა კლასიკურია - როცა ერთი წევრი ვიცით, მეორეს ძებნა მარტივდება.

## ხშირი შეცდომები

- nested loop-ების გამოყენება, რაც O(n²)-ს იძლევა და TLE-ს აგდებს.
- long long-ის დავიწყება, როცა რიცხვები 10⁹-ის ფარგლებშია.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ, s = 10 და მასივის ელემენტია 3. ჩვენ ვეძებთ 7-ს მარჯვენა მხარეს. თუ binary search-ით 7-ს ვიპოვით, პასუხი YES-ია, თუ ციკლი მორჩა და ვერაფერი ვნახეთ, ვბეჭდავთ NO-ს.
