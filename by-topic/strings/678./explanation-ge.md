# განმარტება — 678. დაშიფრული წერილი

## მიდგომა

იდეა მარტივია: სტრიქონში სადაც ასოებს ვხვდებით, უბრალოდ გვერდი უნდა ავუაროთ და მხოლოდ ციფრების ბლოკები ამოვიღოთ. ორმაგი ციკლით მივყვები სტრიქონს და როცა ციფრი შემხვდება, მანამ ვაგროვებ სანამ ისევ ასო არ წამოვა. ეს ყველაფერი `std::set`-ში ჩავყარე, რომ დუბლიკატები თავისით გაიფილტროს. ბოლოს `st.size()-1` იმიტომ მიწერია, რომ როცა ციფრები არ არის, ცარიელ სტრიქონსაც ამატებს სეტში და ეგ ერთი ზედმეტი ელემენტი უნდა მოვაშორო.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- set-ის გამოყენება იდეალურია, რადგან დუბლიკატების კონტროლზე ფიქრი აღარ გიწევს.
- ASCII კოდებით მანიპულაცია (s[j] < 97) ძალიან აჩქარებს შემოწმებას, რადგან ციფრები ლათინურ პატარა ასოებზე წინ მოდის.

## ხშირი შეცდომები

- ცარიელი სტრიქონის გათვალისწინება სეტში, რაც საბოლოო პასუხს ერთით ზრდის.
