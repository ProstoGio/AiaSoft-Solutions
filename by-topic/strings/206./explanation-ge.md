# განმარტება — 206. კენტები ლუწების წინააღმდეგ

## მიდგომა

აქ მთავარი ხრიკი ისაა, რომ 10^100-ს ვერცერთი სტანდარტული ტიპი ვერ დაიჭერს, ამიტომ რიცხვს როგორც სტრიქონს ისე ვკითხულობთ. თითოეულ სიმბოლოს ვამოწმებთ, გადაგვყავს ციფრად და ვნახულობთ ლუწია თუ კენტი. ბოლოს უბრალოდ ვადარებთ ორი ცვლადის მნიშვნელობას. სულ ესაა, მეტი არაფერი უნდა.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- როცა ხედავ ასეთ უზარმაზარ რიცხვს, მაშინვე სტრიქონზე გადადი, არც სცადო int-ებით წვალება.
- ASCII-დან რომ რიცხვი მიიღო, '0'-ის გამოკლება ყველაზე მარტივი და კლასიკური მეთოდია.

## ხშირი შეცდომები

- long long-ის გამოყენება, რაც ამხელა რიცხვისთვის არაფერს მოგცემს და overflow მოგივა.
