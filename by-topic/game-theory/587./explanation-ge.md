# განმარტება — 587. თამაში გადანაცვლებაზე

## მიდგომა

რადგან n მაქსიმუმ 15-ია, brute-force-ს არ უნდა შევეშინდეთ. bitmask-ით ძალიან მარტივად შეგვიძლია ვმართოთ, რომელი ელემენტი გვაქვს დარჩენილი. თამაში არის კლასიკური ნიმუში, სადაც თუ მიმდინარე მდგომარეობა (mask) უკვე ზრდადია, მაშინ მოთამაშე, რომელსაც სვლა უწევს, ავტომატურად აგებს, რადგან თამაში უკვე დამთავრებულია. დანარჩენ შემთხვევებში კი ვცდილობთ ნებისმიერი ელემენტის მოშორებას და ვამოწმებთ, იქნებ მოწინააღმდეგისთვის ისეთი მდგომარეობა დაგვრჩეს, საიდანაც ის წააგებს.

## სირთულე

| | |
|-|--|
| დრო | O(T * n * 2^n) |
| მეხსიერება | O(2^n) |

## მთავარი დაკვირვებები

- bitmask-ი აქ იდეალურია, იმიტომ რომ 2^15 სულ რაღაც 32,768 მდგომარეობაა, რაც მეხსიერებაშიც მშვენივრად ეტევა და დროშიც.
- გახსოვდეთ, რომ recursion-ის დროს memoization აუცილებელია, თორემ exponential დროში ჩავრჩებით.

## ხშირი შეცდომები

- dp მასივის გასუფთავება ყოველი test case-ისთვის. თუ ამას არ გააკეთებ, ძველი შედეგები ახალ ტესტებს არასწორად დაემთხვევა.
- თამაშის პირობის არასწორად აღქმა: ზრდადი მიმდევრობა ნიშნავს რომ აღარაფერი გაქვს გასაკეთებელი და შესაბამისად წაგებული ხარ.

## ნაბიჯ-ნაბიჯ ახსნა

წარმოიდგინე n=3 და [3, 2, 1]. ვიწყებთ 111 მასკით. ვცდი 3-ის მოშორებას, დავრჩები [2, 1]-ით (mask 011). ეს არ არის ზრდადი, ამიტომ რეკურსიაში უფრო ღრმად მივდივარ. როგორც კი დავრჩები ვთქვათ [1]-ზე ან [2]-ზე, რაც უკვე ზრდადია, ფუნქცია დამიბრუნებს 0-ს და მე ამოვალ ზემოთ იმ დასკვნამდე, რომ წინა ნაბიჯი იყო მომგებიანი.
