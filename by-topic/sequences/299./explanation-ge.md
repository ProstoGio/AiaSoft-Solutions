# განმარტება — 299. კიბე და ბურთულა

## მიდგომა

ეს ტიპური DP-ა, პრაქტიკულად ფიბონაჩის მიმდევრობის ანალოგიური, ოღონდ აქ წინა სამი საფეხურის ჯამს ვიღებთ. ლოგიკა მარტივია: რადგან ერთ, ორ ან სამ საფეხურს ვხტებით, შესაბამისად N-ურ საფეხურზე მოსახვედრად გზების რაოდენობა არის N-1, N-2 და N-3 საფეხურების ჯამი. თავიდანვე `d` მასივი შევქმენი და ბაზისური შემთხვევები (1, 2, 3) პირდაპირ გავწერე, მერე კი უბრალოდ ციკლით ავაწყვე დანარჩენი.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- ფორმულა არის d[i] = d[i-1] + d[i-2] + d[i-3], რაც პირდაპირ გამომდინარეობს პირობიდან.
- პირველი სამი საფეხური ხელით უნდა განსაზღვრო, თორემ ციკლი შეცდომას ამოაგდებს.

## ხშირი შეცდომები

- პატარა N-ების გაუთვალისწინებლობა - თუ `n=1` შემოვიდა და კოდი 4-დან იწყებს ციკლს, შეიძლება რაღაც არასწორი მნიშვნელობა ამოაგდოს.
