# განმარტება — 376. კომპონენტები

## მიდგომა

ეს ამოცანა პირდაპირ გვეუბნება, რომ DSU (Disjoint Set Union) უნდა გამოვიყენოთ. როცა წიბოს ვამატებთ, უბრალოდ ვაერთიანებთ ორ კომპონენტს და მათი ზომები უნდა დავაჯამოთ. იმისთვის, რომ ყოველ ჯერზე ზრდადობით დავბეჭდოთ ზომები, `std::multiset` გამოვიყენე, რადგან ის ავტომატურად ინახავს ელემენტებს სორტირებულად და ოპერაციებიც სწრაფია.

## სირთულე

| | |
|-|--|
| დრო | O(M * (α(N) + N)) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- multiset-ის გამოყენება იდეალურია, როცა კომპონენტების ზომები ხშირად იცვლება და ყოველ ჯერზე დალაგებული გვინდა.
- DSU-ს სტანდარტული იმპლემენტაცია path compression-ით საკმარისია, რადგან N=1000-ია და დროში უპრობლემოდ ვეტევით.

## ხშირი შეცდომები

- ყველაზე ხშირი შეცდომა std::set-ის გამოყენებაა, რომელიც ერთნაირ ზომებს 'ჭამს' და საბოლოო პასუხში ზომებს გვაკლებს.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ გვაქვს 3 წვერო: {1}, {2}, {3}. სტრიქონში გვაქვს {1, 1, 1}. შემოდის წიბო (1,2) -> ვშლით ორ 1-ს, ვამატებთ 2-ს. შედეგი: {1, 2}.
