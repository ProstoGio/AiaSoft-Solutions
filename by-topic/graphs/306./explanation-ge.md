# განმარტება — 306. მარიამის დაბადების დღე

## მიდგომა

ეს ამოცანა ტიპური 'Bipartite Graph'-ის შემოწმებაა. რადგან ორ მაგიდასთან უნდა გავანაწილოთ მეგობრები, ეს ნიშნავს, რომ ყოველი უთანხმოება წიბოა და თუ გრაფი 2-ად იღებება, მაშინ პასუხი 'YES'-ია. DFS-ით მივყვები თითოეულ კომპონენტს და ფერებს ვუცვლი მეზობლებს - თუ შემთხვევით მეზობელს უკვე იგივე ფერი დახვდა, რაც ჩემს ნოდს აქვს, ე.ი. კონფლიქტი გარდაუვალია და 'NO'-ს ვაბრუნებ.

## სირთულე

| | |
|-|--|
| დრო | O(N + M) |
| მეხსიერება | O(N + M) |

## მთავარი დაკვირვებები

- გრაფი თუ 2-ად იღებება, ესე იგი შესაძლებელია დაყოფა. თუ არა - მარიამის მეგობრებს ვერ დავსვამთ მშვიდობიანად.
- მნიშვნელოვანია, რომ `used` მასივი გამოვიყენოთ, რათა ციკლებში არ ავირიოთ და ყველა კომპონენტი შევამოწმოთ.

## ხშირი შეცდომები

- არასრული გრაფის შემთხვევაში მხოლოდ ერთი კომპონენტის შემოწმება. ყოველთვის გაიარე ყველა ნოდი `for` ციკლით, თუ `used[i]` ნულია.
- ხელით წვალება რთული backtracking-ით, როცა უბრალო DFS-ით ეგრევე ჩანს ფერის კონფლიქტი.

## ნაბიჯ-ნაბიჯ ახსნა

მაგალითად, თუ 1-2-ს შორის უთანხმოებაა და 3-4-ს შორის: DFS იწყებს 1-ით (ფერავს 1-ით), გადადის 2-ზე (ფერავს 0-ით). მერე ციკლი გადადის 3-ზე, რადგან ის ჯერ არ გვაქვს ნანახი, და ფერავს მას 1-ით, ხოლო 4-ს 0-ით. ყველაფერი რიგზეა, გრაფი ბიპარტიტულია.
