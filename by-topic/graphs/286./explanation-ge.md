# განმარტება — 286. სინონიმები

## მიდგომა

სინონიმების ჯაჭვური ბუნება პირდაპირ მიგვანიშნებს გრაფებზე და კომპონენტების შეკვრაზე, ამიტომ DSU-ს გამოყენება ყველაზე ადეკვატური გზაა. თავიდან თითოეულ სიტყვას ცალკე ჯგუფად ვთვლით, მერე კი სინონიმური წყვილების მოსვლისას union-ით ვაერთიანებთ. ბოლოს უბრალოდ ვითვლით, რამდენი 'root'-ი გვაქვს და რომელი ჯგუფია ყველაზე მსუყე.

## სირთულე

| | |
|-|--|
| დრო | O(N²) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- თუ A=B და B=C, ავტომატურად A=C; ეს არის ტიპური connected components ამოცანა.
- DSU-ს find_set-ი ყოველთვის უნდა გამოვიძახოთ ბოლოს, რომ ყველა ელემენტი თავის საბოლოო root-ზე დავსვათ.

## ხშირი შეცდომები

- მხოლოდ პირდაპირი კავშირების გათვალისწინება და ტრანზიტულობის დავიწყება.
- ჯგუფიდან ჯგუფში გადათვლისას parent-ების სწორად არ გათანაბრება.
