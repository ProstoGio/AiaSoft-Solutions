# განმარტება — 472. ოთახის ფართობი

## მიდგომა

ეს ამოცანა რეალურად გრაფია, სადაც წერტილები კვანძებია, ხოლო მათი მეზობლობა — წიბოები. ყველაზე მარტივი გზა, რაც თავში მომივიდა, DFS-ის გამოყენებაა: როგორც კი '.'-ს ვიპოვი, რომელიც ჯერ არ მინახავს, ვიწყებ 'flood fill'-ს. ყოველი ოთახის ზომას ცალკე ვითვლი, ვინახავ მაქსიმალურს და გადავდივარ შემდეგზე.

## სირთულე

| | |
|-|--|
| დრო | O(N²) |
| მეხსიერება | O(N²) |

## მთავარი დაკვირვებები

- matrix-ში საზღვრების შემოწმება აუცილებელია, თორემ მარტივად 'გაგექცევა' ინდექსი და სეგმენტაციის შეცდომას ამოაგდებს.
- used მასივის გამოყენება იმიტომაა კრიტიკული, რომ მეორედ არ შევიდეთ უკვე დათვლილ ოთახში და O(N²) დროში ჩავეტიოთ.

## ხშირი შეცდომები

- ბრმა წერტილები საზღვრებთან: ხშირად ავიწყდებათ, რომ i-1 ან j+1 შეიძლება მატრიცის გარეთ გავიდეს.
- ans ცვლადის ნულზე ჩამოყრა დაგავიწყდეთ შემდეგი ოთახისთვის.

## ნაბიჯ-ნაბიჯ ახსნა

წარმოიდგინე მატრიცა: მარყუჟით მივყვები თითოეულ უჯრას, თუ 'used[i][j]' არის 0 და წერტილია, ვუშვებ DFS-ს. DFS-ი რეკურსიულად დადის ოთახის გარშემო, ყველას ნიშნავს როგორც 'used=1'-ს და ზრდის `ans`-ს. როგორც კი ფუნქცია დაასრულებს მუშაობას, `ans`-ში მიდევს კონკრეტული ოთახის ფართობი.
