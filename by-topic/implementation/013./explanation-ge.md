# განმარტება — 013. მოქმედებები ვექტორზე

## მიდგომა

მოკლედ რომ ვთქვათ, ეს ამოცანა პირდაპირ C++-ის std::vector-ის მეთოდების პრაქტიკაა. რადგან Q მხოლოდ 100-ია, არაფერი რთული ოპტიმიზაცია არ სჭირდება, უბრალოდ ვკითხულობთ ბრძანებას და შესაბამის ფუნქციას ვიძახებთ. მთავარი აქ არის ინდექსებთან ფრთხილად ყოფნა, რადგან ნიკა 1-დან ათვლის, ვექტორი კი - 0-დან. .insert() და .erase() ავტომატურად გააკეთებენ ყველაფერს, შენ მხოლოდ პოზიციები უნდა მიუთითო სწორად.

## სირთულე

| | |
|-|--|
| დრო | O(Q * N) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- პრობლემაში პოზიციები 1-დან იწყება, ამიტომ v.begin() + p - 1 დაწერით ზუსტად იმ ინდექსს ვწვდებით, რაც გვინდა.
- erase-ის დროს მეორე არგუმენტი სეგმენტის ბოლო წერტილია, ამიტომ p2-ს არ ვაკლებთ 1-ს, რომ დიაპაზონში ზუსტად მოხვდეს.

## ხშირი შეცდომები

- ინდექსების არევა, როცა p1-დან p2-მდე წაშლას აკეთებ.
