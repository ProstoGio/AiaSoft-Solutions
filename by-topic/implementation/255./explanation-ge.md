# განმარტება — 255. მასივის დალაგება - 2

## მიდგომა

ეს ამოცანა სტანდარტული დალაგების საკითხია. რადგან N=10⁵-ია, O(n²) სირთულის ალგორითმები (მაგალითად bubble sort) ვერ გაივლის, ამიტომ C++-ის სტანდარტული sort ფუნქცია გამოვიყენე. ის მუშაობს O(n log n) დროში, რაც სავსებით საკმარისია 1 წამიან ლიმიტში. კოდი უბრალოდ კითხულობს მასივს, ალაგებს და ბეჭდავს.

## სირთულე

| | |
|-|--|
| დრო | O(n log n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- std::sort იყენებს Introsort-ს, რომელიც ამ ტიპის ამოცანებისთვის ოპტიმალურია.
- 10⁵ ელემენტის წაკითხვა-დაბეჭდვისას `ios::sync_with_stdio(0)`-ის გამოყენება ხანდახან სასარგებლოა, თუმცა აქ ამის გარეშეც მშვენივრად მუშაობს.

## ხშირი შეცდომები

- Bubble sort-ის ან Insertion sort-ის გამოყენება, რაც Time Limit Exceeded-ს გამოიწვევს.
- ჰედერის (header) დაკარგვა, რის გარეშეც sort ფუნქცია არ იმუშავებს.

## ნაბიჯ-ნაბიჯ ახსნა

ჯერ ვქმნი vector-ს N ზომით, შემდგომ ციკლით შემაქვს მასივში რიცხვები. sort(a.begin(), a.end()) ბრძანებით მასივი ლაგდება არაკლებადი თანმიმდევრობით, ბოლოს კი უბრალოდ ვატრიალებ ციკლს და ეკრანზე გამომაქვს შედეგი.
