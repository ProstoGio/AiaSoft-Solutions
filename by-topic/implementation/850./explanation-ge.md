# განმარტება — 850. მინიმალური კენტი

## მიდგომა

მოკლედ, ლოგიკა მარტივია: მასივში ვეძებ პირველივე კენტ რიცხვს. როგორც კი ვიპოვი, ვნიშნავ მას როგორც 'დროებით მინიმუმს' და ვიწყებ დანარჩენი კენტი რიცხვების შედარებას მასთან. თუ უფრო პატარა კენტი შემხვდა, მინიმალურს ვანახლებ. თუ მთელი მასივი დავათვალიერე და კენტი ვერ ვიპოვე, პროგრამა ავტომატურად ბეჭდავს -1-ს.

## მთავარი დაკვირვებები

- პირველივე ნაპოვნი კენტი რიცხვი გვაძლევს საწყის წერტილს შედარებისთვის.
- თუ მასივში არცერთი კენტი არაა, კოდი არცერთხელ შევა 'if' ბლოკში და პირდაპირ -1-ს გამოიტანს.

## ხშირი შეცდომები

- პროგრამის გაჩერება მას შემდეგ, რაც პირველ კენტ რიცხვს ვიპოვით, ნაცვლად იმისა, რომ მთელი მასივი შევამოწმოთ მინიმალურისთვის.
