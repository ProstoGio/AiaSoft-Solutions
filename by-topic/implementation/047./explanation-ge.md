# განმარტება — 047. ლამაზი წინადადება

## მიდგომა

სიმბოლოებს სათითაოდ გავუყვები და შევამოწმებ, უნდა იყოს თუ არა დიდი ასო. თუ სიმბოლო წინადადების დასაწყისშია ან წინ 'სპეისი' უძღვის, ავტომატურად ვამცირებ მის ASCII კოდს 32-ით, რომ დიდი გახდეს. ყველა სხვა შემთხვევაში, თუ ასო დიდია, პირიქით, ვუმატებ 32-ს, რომ დავაპატარაო. ეს მიდგომა საშუალებას მაძლევს, მთელი სტრიქონი ერთი გავლით დავალაგო.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- ASCII კოდების გამოყენება ყველაზე სწრაფი გზაა ასოების რეგისტრის შესაცვლელად.
- სპეისების სწორად აღქმა წყვეტს სიტყვების საზღვრებს.
- პირველი სიმბოლოს ცალკე შემოწმება გვაზღვევს ინდექსების არევისგან.

## ხშირი შეცდომები

- სიტყვის შუაში არსებული დიდი ასოების პატარებად არ გადაქცევა.
- getline-ის ნაცვლად cin-ის გამოყენება, რაც სტრიქონს მხოლოდ პირველ სპეისამდე წაიკითხავდა.

## ნაბიჯ-ნაბიჯ ახსნა

getline-ით ვკითხულობ მთლიან წინადადებას. ციკლში ვამოწმებ: თუ i=0 ან s[i-1] არის ' ', მაშინ ვაკეთებ `s[i]-=32` თუ პატარაა. ხოლო თუ შუა ასოა, ვაკეთებ `s[i]+=32` იმ შემთხვევაში თუ ის უკვე დიდი ასოა. ბოლოს უბრალოდ ვბეჭდავ შეცვლილ სტრიქონს.
