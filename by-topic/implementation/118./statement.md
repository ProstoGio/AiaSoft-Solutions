# Problem Statement — 118. ჯვარი

## Task

მოცემულია მართკუთხა A მატრიცა, რომელიც მხოლოდ 0 - ებისა და 1 - ებისაგან შედგება (ბინარული მატრიცა). ერთიანების ჯვარი ცენტრით ARS უჯრაში ვუწოდოთ ისეთ კონფიგურაციას, რომლისთვისაც სრულდება პირობები:

1. არსებობს ისეთი P და Q ინდექსები, სადაც P < Q, რომ ყველა მიმდევრობით აღებული ციფრი ARP - დან ARQ - მდე მხოლოდ 1 - იანებია;
2. არსებობს ისეთი C და D ინდექსები, სადაც C < D, რომ ყველა მიმდევრობით აღებული ციფრი ACS - დან ADS - მდე მხოლოდ 1 - იანებია;
3. P < S < Q  და  C < R < D

მაგალითად, ქვემოთ ნაჩვენებ სურათზე მხოლოდ A) და B) შემთხვევებში გვაქვს ერთიანების ჯვარი.

`~https://storage.googleapis.com/problems_api_photos/images/problem118.png`

დაწერეთ პროგრამა, რომელიც დაადგენს ერთიანების რა მაქსიმალურ რაოდენობას შეიცავს ჯვარი ცენტრით ARS უჯრაში. თქვენ უფლება გეძლევათ არაუმეტეს ერთი 0 შეცვალოთ 1 - ით.

## Input

სტანდარტული შეტანის პირველ სტრიქონში მოცემულია ერთი ჰარით გამოყოფილი ორი მთელი დადებითი M და N რიცხვი (1≤ M, N ≤ 1000) – A მატრიცის სტრიქონებისა და სვეტების რაოდენობები შესაბამისად. მომდევნო M რაოდენობის სტრიქონიდან თითოეულში გამყოფების გარეშე ჩაწერილია N რაოდენობის ორობითი ციფრი. შემდეგი სტრიქონი შეიცავს ერთ მთელ დადებით T რიცხვს (1≤ T ≤ 10000) - პოტენციური საძიებელი ჯვრების რაოდენობას. მომდევნო T რაოდენობის სტრიქონიდან თითოეულში კი ჩაწერილია თითო ჰარით გამოყოფილი ორი მთელი Ri და Si  რიცხვი - i - ური პოტენციური საძიებელი ჯვრის ცენტრის კოორდინატები (სტრიქონისა და სვეტის ნომრები).

## Output

სტანდარტული გამოტანის თითოეულ სტრიქონში თითოეული ნაპოვნი ჯვრისათვის უნდა გამოიტანოთ ერთი მთელი რიცხვი - მასში ერთიანების მაქსიმალური რაოდენობა. თუ ასეთი ჯვარი ცენტრით მოცემულ წერტილში არ არსებობს, მაშინ შესაბამის სტრიქონში უნდა ჩაიწეროს -1. 
0 - ის 1 - ით შეცვლა შესაძლებელია ყოველ კონკრეტულ შემთხვევაში (ყოველი კონკრეტული ჯვრის ძიებისას).
