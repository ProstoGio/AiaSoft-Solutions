# განმარტება — 366. რამდენი ნულია?

## მიდგომა

აქ იდეა ძალიან მარტივია: რეკურსიით ვშლი რიცხვს ბოლო ციფრების მიხედვით. მთავარი ყურადღება მქონდა იმაზე, რომ n=0-ისთვის პასუხი 1 უნდა იყოს, სხვა ერთნიშნა რიცხვებისთვის კი - 0. ყოველ ნაბიჯზე ვამოწმებ ნაშთს 10-ზე, თუ 0-ია, მაშინ ვუმატებ ერთს და გადავდივარ n/10-ზე. არ მინდოდა სტრინგად გადაყვანა და მერე თვლა, ესე პირდაპირ მათემატიკურად უფრო ლამაზად იწერება.

## სირთულე

| | |
|-|--|
| დრო | O(log n) |
| მეხსიერება | O(log n) |

## მთავარი დაკვირვებები

- 0-ის ცალკე დამუშავებაა საჭირო, თორემ თუ უბრალოდ ნაშთებით ვიმუშავებდით, 0-ს საერთოდ ვერ დავიჭერდით.
- რეკურსია აქ იდეალურია, რადგან ყოველ ეტაპზე რიცხვი 10-ჯერ მცირდება, რაც log n სირთულეს გვაძლევს.

## ხშირი შეცდომები

- 0-ის შემთხვევის გამოტოვება, რის გამოც ზოგიერთმა შესაძლოა 0 დააბრუნოს, როცა სინამდვილეში 1-ია.
