# განმარტება — 265. უდიდესი კომპონენტი

## მიდგომა

ამ ამოცანაში ყველაზე მოსახერხებელი გზა Disjoint Set Union-ის, ანუ DSU-ს გამოყენებაა. თავიდან ყველა წვერო თავის თავის 'მამად' ითვლება, მერე კი თითოეულ წიბოზე union-ს ვაკეთებთ და ამით მეზობელ წვეროებს ერთ ჯგუფში ვაერთიანებთ. ბოლოს, უბრალოდ ვამოწმებ თითოეული წვეროს საბოლოო root-ს, ვითვლი რამდენი წვერო ეკუთვნის თითოეულ ჯგუფს და მაქსიმუმს ვიმახსოვრებ. მარტივი და ეფექტურია, თან კოდიც მცირე გამოდის.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- DSU-ს გამოყენება გრაფიდან კომპონენტების გამოსაყოფად ყველაზე სუფთა გზაა, აღარ გიწევს BFS ან DFS-ის წერა.
- საბოლოოდ, როცა ყველა წიბოს დავამუშავებთ, მნიშვნელოვანია თითოეულ წვეროზე find_set(i) გავუშვათ, რომ parent-ი პირდაპირ root-ზე მიუთითებდეს.

## ხშირი შეცდომები

- ზოგჯერ ავიწყდებათ იმ შემთხვევის გათვალისწინება, როცა გრაფში იზოლირებული წვეროებია, თუმცა DSU-ს შემთხვევაში ეს ავტომატურად წყდება.
