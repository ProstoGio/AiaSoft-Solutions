# განმარტება — 532. მოხეტიალე ცხენი

## მიდგომა

მოკლედ რომ ვთქვათ, ეს სტანდარტული BFS-ია. ჩვეულებრივი ცხენის სვლების მაგივრად, აქ უჯრაზე დაწერილი სიმბოლო გვზღუდავს. თუ უჯრაზე 'U' წერია, მხოლოდ იმ სვლებს განვიხილავთ, რომლებიც უფრო მაღლა (უფრო პატარა სტრიქონზე) გადაგვიყვანს. BFS ზუსტად იმიტომ გამოგვადგება, რომ ყველაზე მოკლე გზას ვეძებთ და როგორც კი (N, M)-ს მივაღწევთ, ეგ იქნება პასუხი.

## სირთულე

| | |
|-|--|
| დრო | O(N * M) |
| მეხსიერება | O(N * M) |

## მთავარი დაკვირვებები

- გრაფი სტატიკური არაა; თითოეულ უჯრაზე სვლები დინამიკურად იფილტრება სიმბოლოს მიხედვით.
- არ დაგავიწყდეთ საზღვრების (1-დან N/M-მდე) შემოწმება ყოველი ნაბიჯისთვის, თორემ segmentation fault გარანტირებულია.

## ხშირი შეცდომები

- used მასივის განახლების დავიწყება, რამაც შეიძლება უსასრულო ციკლში ჩაგვაგდოს.
- სიმბოლოების ლოგიკის არასწორი ინტერპრეტაცია (მაგალითად, 'U' თუ მარტო სვეტის ცვლილებას გულისხმობს თუ მარტო სტრიქონის).

## ნაბიჯ-ნაბიჯ ახსნა

ვიწყებთ (1, 1)-დან. თუ ვდგავართ უჯრაზე, სადაც 'R'-ია, ვამოწმებთ 8 შესაძლო სვლიდან მხოლოდ იმ 4-ს, რომელიც მარჯვნივ გადაგვანაცვლებს. ვინახავთ დისტანციას d[i][j] მასივში და ვაგრძელებთ სანამ მიზანს არ მივადგებით.
