# განმარტება — 330. კენტი რიცხვები 3

## მიდგომა

თავიდანვე ვამოწმებ რომელი რიცხვია უფრო დიდი, რომ არ ვიწვალო და range ყოველთვის დალაგებული მქონდეს. თუ 'a' მეტია 'b'-ზე, უბრალოდ ვცვლი მათ ადგილებს. ამის შემდეგ, თუ 'a' კენტია, პირდაპირ ვბეჭდავ და 'a'-ს ვაზღვევ, რომ მერე მარტივი 'while'-ით გადავუყვე ნაბიჯებით. ასე აღარ ვწვალობ if-ებით ლუპის შიგნით.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- swap-ი აუცილებელია, თორემ input-ის არეულობა ლოგიკას დაგიშლის.
- პირველივე რიცხვის დამუშავება ლუპის გარეთ კოდს უფრო სუფთას და წასაკითხს ხდის.

## ხშირი შეცდომები

- იმის დავიწყება, რომ a შეიძლება მეტი იყოს b-ზე.
