# განმარტება — 755. 2-ის ახარისხება

## მიდგომა

ეს ამოცანა რეკურსიის დასამუღამებლად იდეალურია. ლოგიკა მარტივია: 2^n არის 2 გამრავლებული 2^(n-1)-ზე. შესაბამისად, ბაზისური შემთხვევა არის n=0, სადაც პასუხი 1-ია, დანარჩენ შემთხვევებში კი ფუნქციას თავისივე თავი გამოჰყავს n-1-ით. ბიტით დაწევა (bitwise shift) უფრო სწრაფი იქნებოდა, მარა ამ დონეზე რეკურსია საკმარისია.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- ბაზისური შემთხვევის (n=0) სწორად განსაზღვრა ყველაზე მნიშვნელოვანია, თორემ რეკურსია უსასრულოდ გაგრძელდება.
- რეკურსიული გამოძახებები მანამ მიდის, სანამ 0-ს არ მივაღწევთ, მერე კი უკან ბრუნდება და ამრავლებს მნიშვნელობებს.

## ხშირი შეცდომები

- n=0 შემთხვევის გამოტოვება, რაც stack overflow-ს იწვევს.
