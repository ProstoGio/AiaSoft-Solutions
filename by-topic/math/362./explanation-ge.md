# განმარტება — 362. მკაცრი ლუწი რიცხვი

## მიდგომა

აქ მთავარი ხრიკი ისაა, რომ რიცხვი ბოლოდან უნდა დავშალოთ ციფრებად. `n % 10`-ით ყოველ ჯერზე ვიღებ ბოლო ციფრს და ვამოწმებ კენტია თუ არა. თუ რომელიმე ციფრი კენტი აღმოჩნდა, პასუხად პირდაპირ 'NO'-ს ვბეჭდავ და ვწყვეტ მუშაობას. თუ მთელი რიცხვი ისე გადავიარე, რომ კენტი ვერ ვიპოვე, ე.ი. ყველაფერი ლუწია და 'YES'-ს ვაბრუნებ.

## სირთულე

| | |
|-|--|
| დრო | O(log n) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- რიცხვის ციფრებად დაშლა % 10-ით და `/ 10`-ით ყველაზე სწრაფი მეთოდია, ციკლის რაოდენობა ციფრების რაოდენობის ტოლია.
- საკმარისია ერთი კენტი ციფრი ვიპოვოთ, რომ პირობა დაირღვეს, ამიტომ არ გვჭირდება რიცხვის ბოლომდე დამუშავება.

## ხშირი შეცდომები

- მთლიანი რიცხვის ლუწობაზე შემოწმება ნაცვლად თითოეული ციფრისა.
