# განმარტება — 458. პასკალის სამკუთხედი

## მიდგომა

პასკალის სამკუთხედი ტიპური DP ამოცანაა, სადაც თითოეული უჯრა თავის ზემოთ მდგომი ორი რიცხვის ჯამია. არ მინდოდა კომბინატორიკით თავის ტკივილი, ამიტომ უბრალოდ 2D მასივი გამოვიყენე. კიდეები ერთიანებით შევავსე და შუა ნაწილი მარტივი ციკლით დავთვალე. N+1 ზომის მასივი იმიტომ ავიღე, რომ ინდექსებთან ჩალიჩი არ დამჭირვებოდა და საზღვრები არ დამრღვეოდა.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n²) |

## მთავარი დაკვირვებები

- ყოველი უჯრა (i, j) არის წინა სტრიქონის იმავე და მის მარცხენა ინდექსზე მყოფი რიცხვების ჯამი.
- მასივის ზომის N+1-მდე გაზრდა გვიცავს ეგრეთ წოდებული 'out of bounds' შეცდომებისგან.

## ხშირი შეცდომები

- ფაქტორიალების გამოყენება არ ვარგა, რადგან N-ის ზრდასთან ერთად რიცხვები ძალიან სწრაფად სცდება int-ის ზღვარს.
