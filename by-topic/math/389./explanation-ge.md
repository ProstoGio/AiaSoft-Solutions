# განმარტება — 389. სრულყოფილი რიცხვი

## მიდგომა

სრულყოფილი რიცხვის პოვნა მარტივია: უბრალოდ უნდა შევკრიბოთ ყველა გამყოფი n-ის გარდა და ვნახოთ, მივიღებთ თუ არა ისევ n-ს. რადგან n მხოლოდ 10⁵-მდეა, პირდაპირი ციკლი თავისუფლად გაქაჩავს და არაფერი ზედმეტი ოპტიმიზაცია არ სჭირდება. კოდში უბრალოდ 1-დან n-1-მდე ვტრიალებთ, modulo-თი ვამოწმებთ გამყოფებს და ვუმატებთ ჯამს.

## სირთულე

| | |
|-|--|
| დრო | O(n) |
| მეხსიერება | O(1) |

## მთავარი დაკვირვებები

- განმარტების მიხედვით, საკუთარი თავი არ ითვლება, ამიტომ ციკლი n-ის ჩათვლის გარეშე უნდა დავასრულოთ.
- 10⁵-ისთვის O(n) სირთულე წამზე ნაკლებში იდეალურად მუშაობს.

## ხშირი შეცდომები

- ციკლის 0-დან დაწყება, რაც runtime error-ს გამოიწვევს division by zero-ს გამო.
- შემოწმებისას თვითონ რიცხვის დამატება ჯამში.
