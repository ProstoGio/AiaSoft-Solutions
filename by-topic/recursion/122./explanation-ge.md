# განმარტება — 122. სტრიქონი

## მიდგომა

აქ ჩვეულებრივი `for` ციკლის ნაცვლად რეკურსია გამოვიყენე, უბრალოდ იმიტომ რომ კოდი უფრო საინტერესო ყოფილიყო. ლოგიკა მარტივია: ფუნქცია ამოწმებს სტრიქონის პირველ სიმბოლოს, თუ ციფრია, ითვლის როგორც ერთს და აგრძელებს დარჩენილ სტრიქონზე მუშაობას. ასე ნელ-ნელა, `erase`-ით ვამცირებ სტრიქონს, სანამ ცარიელი არ დარჩება.

## სირთულე

| | |
|-|--|
| დრო | O(n²) |
| მეხსიერება | O(n) |

## მთავარი დაკვირვებები

- `x.erase(0, 1)`-ის გამო სტრიქონის ასლი იქმნება ყოველ ჯერზე, რის გამოც გვაქვს O(n²) სირთულე.
- რეკურსიულად პრობლემის პატარა ნაწილებად დაყოფა კაი ვარჯიშია, თუმცა რეალურ ცხოვრებაში ჩვეულებრივი ციკლი ბევრად ოპტიმალურია.

## ხშირი შეცდომები

- ბაზისური შემთხვევის (base case) დავიწყება, რაც რეკურსიის დროს Stack Overflow-ს იწვევს.
