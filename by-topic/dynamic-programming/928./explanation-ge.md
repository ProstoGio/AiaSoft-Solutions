# განმარტება — 928. მიმდევრობები

## მიდგომა

ეს არის ტიპური DP ამოცანა, სადაც სტატუსს განვსაზღვრავთ მიმდევრობის სიგრძით და ბოლო რიცხვის მნიშვნელობით. იდეა მარტივია: თუ ვიცით, რამდენი მიმდევრობა მთავრდება 'j' რიცხვით 'i' სიგრძის შემთხვევაში, შემდეგი ნაბიჯისთვის მხოლოდ იმ 'j-k' და 'j+k' ვარიანტებს ვუყურებთ, რომლებიც 1-სა და m-ს შორის თავსდება. ბოლოს უბრალოდ ვკრებთ ყველა შესაძლო დაბოლოებას n-ური ნაბიჯისთვის და ეგაა.

## სირთულე

| | |
|-|--|
| დრო | O(n * m) |
| მეხსიერება | O(n * m) |

## მთავარი დაკვირვებები

- dp[i][j] ინახავს მიმდევრობების რაოდენობას, რომელთა სიგრძე i-ია და მთავრდება j-ით.
- რადგან n და m საკმაოდ პატარაა, პირდაპირი DP მიდგომა ყველაზე ოპტიმალური და მარტივი გამოსავალია.

## ხშირი შეცდომები

- ზღვრული მნიშვნელობების, ანუ 1-ისა და m-ის, არასწორი გათვალისწინება if-ებში.
- საწყისი მნიშვნელობების (length 1) არასწორად ინიციალიზაცია.

## ნაბიჯ-ნაბიჯ ახსნა

ვთქვათ, n=3, k=2, m=5. თავიდან გვაქვს სიგრძე 1, სადაც ყველა რიცხვს (1, 2, 3, 4) აქვს თითო ვარიანტი. სიგრძე 2-ისთვის ვიანგარიშებთ წინა სტეიჯების ჯამს: მაგალითად, dp[2][3] იქნება dp[1][1] + dp[1][5]-ის (რომელიც 0-ია) ჯამი. ასე მივყვებით ბოლომდე.
