# განმარტება — 452. კვადრების ქვეყანა

## მიდგომა

ეს ტიპური DP ამოცანაა, ზუსტად ისეთი, როგორიც 'Coins'-ის ამოცანებია ხოლმე. უბრალოდ აქ 'მონეტების' ნომინალები არაა მოცემული, ჩვენ თვითონ უნდა მივხვდეთ, რომ ეს კვადრატებია: 1, 4, 9, 16 და ასე შემდეგ. dp[i] მასივში ვინახავთ მინიმალურ რაოდენობას, რაც i-ს ასაწყობად გვჭირდება. თითოეული i-სთვის ვამოწმებთ ყველა შესაძლო კვადრატს (j*j), რომელიც i-ზე ნაკლებია და ვუყურებთ, რომელი კომბინაცია მოგვცემს ყველაზე პატარა შედეგს.

## სირთულე

| | |
|-|--|
| დრო | O(N * sqrt(N)) |
| მეხსიერება | O(N) |

## მთავარი დაკვირვებები

- Greedy აქ არ მუშაობს, რადგან ყოველთვის დიდი კვადრატის გამოკლება არ ნიშნავს მინიმალურ რაოდენობას.
- dp[i] = min(dp[i], dp[i - j*j] + 1) არის ის ფორმულა, რომელიც მთელ ლოგიკას ატარებს.

## ხშირი შეცდომები

- ინდექსების არევა და 'out of bounds' შეცდომები, ან თავიდანვე ძალიან პატარა ზომის მასივის გამოცხადება.

## ნაბიჯ-ნაბიჯ ახსნა

ავიღოთ N=12. პროგრამა აკეთებს შემდეგს: 12-ისთვის ცდის 1, 4 და 9 კვადრატებს. 12-9=3, 12-4=8, 12-1=11. შემდეგ უყურებს უკვე დათვლილ dp[3], dp[8] და dp[11] მნიშვნელობებს და ირჩევს მათგან მინიმალურს, რასაც პლუს 1 ემატება. საბოლოოდ გამოდის 3.
